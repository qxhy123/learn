# 第4章：回溯算法复杂度分析

## 4.1 为什么分析复杂度困难

回溯算法的复杂度分析比普通算法更难，因为：

1. **剪枝效果难以量化**：剪枝后实际访问的节点数远少于理论上界
2. **依赖输入数据**：最坏、平均、最好情况差异巨大
3. **树的形状不规则**：非满树导致精确计算困难

因此，通常给出**最坏情况**的上界。

## 4.2 基本分析框架

```
时间复杂度 = 节点数 × 每个节点的处理时间

节点数 = 决策树中实际访问的节点数
       ≤ 宽度^深度（未剪枝的满树）
```

**常见量级**：

| 问题类型 | 决策树 | 时间复杂度 |
|---------|--------|-----------|
| 子集 | 二叉树，深度 n | O(n × 2^n) |
| 排列 | n 叉树，深度 n | O(n × n!) |
| 组合 C(n,k) | - | O(k × C(n,k)) |
| N 皇后 | - | O(n × n!) |

## 4.3 子集问题 O(n × 2^n)

每个元素有两种选择（选/不选），共 n 个元素：

```
节点总数 = 2^0 + 2^1 + 2^2 + ... + 2^n = 2^(n+1) - 1 = O(2^n)
每个节点的工作 = O(n)（拷贝路径）
总时间 = O(n × 2^n)
空间 = O(n)（递归栈深度）
```

```python
def subsets(nums):  # T = O(n × 2^n), S = O(n)
    result = []
    n = len(nums)
    
    def bt(start, path):
        result.append(path[:])   # O(n) 拷贝
        for i in range(start, n):
            path.append(nums[i])
            bt(i + 1, path)
            path.pop()
    
    bt(0, [])
    return result
```

## 4.4 排列问题 O(n × n!)

每层减少一个选择，形成如下树：

```
第0层：n 个分支
第1层：每个节点 n-1 个分支
...
叶节点数 = n × (n-1) × ... × 1 = n!
内部节点数 ≈ n! × e（自然常数）= O(n!)
每个叶节点工作 = O(n)
总时间 = O(n × n!)
```

```python
def permutations(nums):  # T = O(n × n!), S = O(n)
    result = []
    n = len(nums)
    used = [False] * n
    
    def bt(path):
        if len(path) == n:
            result.append(path[:])  # O(n) 拷贝
            return
        for i in range(n):
            if used[i]:
                continue
            used[i] = True
            path.append(nums[i])
            bt(path)
            path.pop()
            used[i] = False
    
    bt([])
    return result
```

## 4.5 组合问题 O(k × C(n,k))

从 n 个元素中选 k 个：

```
结果数 = C(n,k) = n! / (k! × (n-k)!)
每个结果的工作 = O(k)（拷贝）
总时间 = O(k × C(n,k))

内部节点数 ≈ C(n,0) + C(n,1) + ... + C(n,k) ≤ (k+1) × C(n,k)
```

```
实际数字（n=20, k=10）：
C(20,10) = 184,756
时间 ≈ 10 × 184,756 = 1,847,560  ← 可接受

若 k = n（全选），退化为子集：O(n × 2^n)
```

## 4.6 N 皇后的精确分析

N 皇后没有简单的闭合公式，给出各 N 的实际解数：

| N | 解的数量 | 搜索节点数（近似） |
|---|---------|----------------|
| 1 | 1 | 1 |
| 4 | 2 | 8 |
| 8 | 92 | ~2,000 |
| 12 | 14,200 | ~3,000,000 |
| 15 | 2,279,184 | ~300,000,000 |

理论上界 N × N!（约束剪枝后实际远小于此）。

## 4.7 剪枝的效果量化

**示例：组合总和**

`candidates = [1,2,3,...,n], target = T`

- 无剪枝：访问 O(n^T) 个节点（每层 n 个选择，最深 T 层）
- 排序+超限剪枝：大幅减少，难以精确计算，但实践中快很多

```python
# 用计数器衡量剪枝效果
visit_count = [0]

def backtrack_with_count(candidates, target):
    candidates.sort()
    
    def bt(start, remaining):
        visit_count[0] += 1  # 计数
        
        if remaining == 0:
            return
        
        for i in range(start, len(candidates)):
            if candidates[i] > remaining:
                break  # 剪枝
            bt(i, remaining - candidates[i])
    
    bt(0, target)
    return visit_count[0]

# 对比
visit_count[0] = 0
backtrack_with_count([1,2,3,4,5], 10)
print(f"有剪枝：{visit_count[0]} 个节点")
# 远少于无剪枝版本
```

## 4.8 空间复杂度

回溯的空间复杂度 = **递归栈深度** + **额外存储**

```
递归栈：O(depth)  -- 最大递归深度
路径存储：O(depth)  -- path 数组
结果存储：O(结果总大小)  -- result 列表

总空间（不含结果）= O(depth)
```

**各问题的栈深度**：
- 子集：O(n)
- 排列：O(n)
- 组合（可重复选）：O(target/min_val)

## 4.9 实践中的时间估算

面试/竞赛中快速估算是否超时（Python ≈ 10^7 ops/sec，C++ ≈ 10^8-9）：

```
n=15, 全排列：15! = 1.3 × 10^12  → 必超时
n=20, 子集：2^20 = 10^6          → 可以
n=25, 子集：2^25 = 3.3 × 10^7   → Python 边界，C++ 可以
n=10, 全排列：10! = 3.6 × 10^6  → 可以
```

| 复杂度 | 可处理的 n | 备注 |
|--------|-----------|------|
| O(2^n) | n ≤ 25 | 子集 |
| O(n!) | n ≤ 10-12 | 排列 |
| O(n × n!) | n ≤ 10 | 输出所有排列 |

## 4.10 最优子结构 vs 最优剪枝

```
问题                 方法
————————————————————————————————
求所有解             → 回溯（必须遍历所有）
求一个解             → 回溯（找到立即返回）
求最优解（无重叠子问题）→ 回溯
求最优解（有重叠子问题）→ 动态规划（比回溯快）
```

**判断是否可以用 DP 代替回溯**：
- 问题是否有最优子结构？
- 子问题是否重叠？
- 是否只需要计数/最值，不需要枚举所有方案？

## 小结

| 问题 | 时间上界 | 实际（有剪枝）|
|------|---------|-------------|
| 子集 | O(n × 2^n) | O(n × 2^n)（无法再减）|
| 排列 | O(n × n!) | O(n × n!)（约束剪枝减常数）|
| 组合总和 | O(n^T/min) | 远小于上界 |
| N皇后 | O(n × n!) | 实际 << 理论上界 |
