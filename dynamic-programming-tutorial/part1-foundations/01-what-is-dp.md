# 第1章：动态规划是什么

## 1.1 从一道题目开始

计算第 n 个斐波那契数：

```
F(0) = 0
F(1) = 1
F(n) = F(n-1) + F(n-2)
```

最直接的递归实现：

```python
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
```

看起来很简单。但计算 `fib(40)` 需要多久？

```python
import time
start = time.time()
print(fib(40))  # 102334155
print(f"耗时: {time.time() - start:.2f}s")  # 约 30+ 秒
```

**为什么这么慢？** 画出递归树：

```
fib(5)
├── fib(4)
│   ├── fib(3)
│   │   ├── fib(2)  ← 重复计算！
│   │   └── fib(1)
│   └── fib(2)      ← 重复计算！
└── fib(3)          ← 重复计算！
    ├── fib(2)      ← 重复计算！
    └── fib(1)
```

`fib(3)` 被计算了 2 次，`fib(2)` 被计算了 3 次，`fib(1)` 被计算了 5 次...

**这就是动态规划要解决的核心问题：重叠子问题（Overlapping Subproblems）。**

---

## 1.2 动态规划的三大要素

动态规划（Dynamic Programming）由数学家 Richard Bellman 在 1950 年代提出。一个问题能用 DP 解决，必须同时满足：

### 要素一：最优子结构（Optimal Substructure）

> 问题的最优解，可以由子问题的最优解构造得到。

**例子**：最短路径问题
- 从 A 到 C 的最短路径，必然经过某个中间点 B
- A→B 那段是 A 到 B 的最短路径
- B→C 那段是 B 到 C 的最短路径

如果不满足这个条件呢？考虑"最长简单路径"问题（不重复访问节点的最长路径）——它**没有**最优子结构，因此无法用 DP。

### 要素二：重叠子问题（Overlapping Subproblems）

> 同样的子问题被反复求解。

这是 DP 与分治法的本质区别：
- **分治法**（如归并排序）：子问题不重叠，每个子问题只解决一次
- **动态规划**：子问题大量重叠，记录结果避免重复计算

### 要素三：无后效性（No Aftereffect）

> 某阶段的状态一旦确定，此后的决策不受该阶段之前历史的影响。

换句话说：**未来只取决于现在，不取决于过去的路径。**

```
合法：dp[i] = dp[i-1] + dp[i-2]
      （只看当前状态，不关心如何到达当前状态）

不合法：dp[i] 的值取决于到达 i 的路径
        （违反无后效性）
```

---

## 1.3 动态规划 vs 其他方法

| 方法 | 核心思想 | 适用场景 |
|------|----------|----------|
| 暴力枚举 | 尝试所有可能 | 数据规模极小 |
| 贪心算法 | 每步选局部最优 | 能证明局部最优=全局最优 |
| 分治法 | 分解为不重叠子问题 | 子问题独立 |
| **动态规划** | **记录子问题结果，避免重复** | **子问题重叠，有最优子结构** |
| 回溯搜索 | 系统枚举+剪枝 | 组合搜索、约束满足 |

## 1.4 DP 的两种实现方式

动态规划有两种等价的实现方式，我们将在后续章节详细讲解：

**方式一：记忆化搜索（Top-Down，自顶向下）**
```python
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
```
- 从原问题出发，递归分解
- 遇到已解决的子问题，直接查表
- **直觉上更自然**

**方式二：制表法（Bottom-Up，自底向上）**
```python
def fib(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
```
- 从最小子问题开始，逐步构建
- 填表顺序需满足依赖关系
- **性能更优，无递归开销**

---

## 1.5 动态规划的本质：数学归纳法

DP 的思维本质是**数学归纳法的计算化**：

1. **定义状态**：`dp[i]` 表示什么？
2. **初始条件**：最小子问题的答案是什么？（类比归纳法的 base case）
3. **转移方程**：如何从小状态推导大状态？（类比归纳步骤）
4. **答案提取**：哪个状态是原问题的答案？

**这四步是动态规划的万能框架。**

## 1.6 一个更直观的类比

想象你在爬楼梯，可以一次走1步或2步。走到第10级台阶有多少种方法？

- 你不需要从头枚举所有路径
- 你只需要知道：**到达第10级 = 从第9级走1步 + 从第8级走2步**
- 这就是"最优子结构"（这里是"计数子结构"）
- 一旦你记住了到达每一级的方法数，就不需要重复计算

```python
def climb_stairs(n):
    if n <= 2:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1  # 到达第1级：1种方法
    dp[2] = 2  # 到达第2级：2种方法（1+1 或 2）
    for i in range(3, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

print(climb_stairs(10))  # 89
```

## 1.7 本章小结

| 概念 | 含义 |
|------|------|
| 重叠子问题 | 同一子问题被多次求解，可以缓存结果 |
| 最优子结构 | 全局最优解由局部最优解构成 |
| 无后效性 | 状态转移只依赖当前状态，不依赖历史路径 |
| 记忆化搜索 | Top-down，递归+缓存 |
| 制表法 | Bottom-up，迭代填表 |
