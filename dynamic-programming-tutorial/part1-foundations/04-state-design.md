# 第4章：状态设计与转移方程——DP 的核心方法论

## 4.1 为什么状态设计最重要

动态规划最难的部分不是写代码，而是**想清楚状态定义**。

同一道题，不同的状态定义会导致：
- 能否正确解题
- 时间复杂度差一个数量级
- 代码复杂度相差数倍

---

## 4.2 状态设计原则

### 原则一：状态必须能描述当前的"处境"

状态 `dp[i]` 应该完整描述：
> "当前处理到了什么位置，做了哪些关键决策，接下来面对什么子问题"

**反例**：`dp[i]` = 前 i 步所有路径的集合  
（过于宽泛，无法直接用于计算）

**正例**：`dp[i]` = 以第 i 个元素结尾的最长递增子序列的长度  
（精确、可计算、满足无后效性）

### 原则二：状态要满足无后效性

**无后效性**：`dp[i]` 的值确定后，从 `dp[i]` 出发的计算不依赖"如何到达 `dp[i]`"的历史。

```python
# 有后效性（错误状态设计）：
# dp[i] = 到达位置 i 时的"最后方向"
# 这不行，因为后续计算需要知道历史路径

# 无后效性（正确状态设计）：
# dp[i] = 到达位置 i 时的最小代价
# 只需知道这个值本身，不需要知道怎么来的
```

### 原则三：状态维度要够但不过多

- 维度过少：无法区分不同的子问题（信息不足）
- 维度过多：状态空间爆炸（复杂度过高）

---

## 4.3 状态设计的五种常见技巧

### 技巧1：添加额外维度

**问题**：状态 `dp[i]` 的转移依赖于某个"上下文"，而这个上下文无法从 `dp[i]` 中推断。

**解决**：增加一个维度来记录这个上下文。

**示例**：股票买卖（LeetCode 122，多次交易）

```python
# 错误尝试：dp[i] = 第 i 天的最大利润
# 问题：不知道第 i 天是否持有股票，无法正确转移

# 正确设计：添加"是否持有股票"维度
# dp[i][0] = 第 i 天不持有股票时的最大利润
# dp[i][1] = 第 i 天持有股票时的最大利润

def max_profit(prices):
    n = len(prices)
    # dp[i][0]: 第i天结束时不持有股票的最大利润
    # dp[i][1]: 第i天结束时持有股票的最大利润
    dp = [[0, 0] for _ in range(n)]
    
    dp[0][0] = 0           # 不持有，利润为0
    dp[0][1] = -prices[0]  # 持有，花费了prices[0]
    
    for i in range(1, n):
        # 不持有：昨天不持有 或 昨天持有今天卖出
        dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i])
        # 持有：昨天持有 或 昨天不持有今天买入
        dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])
    
    return dp[n-1][0]
```

### 技巧2：枚举"最后一步"

**思路**：要求 `dp[i]`，枚举**转移到状态 i 的最后一步**是什么。

**示例**：完全平方数（LeetCode 279）

```python
# dp[n] = 和为 n 所需的最少完全平方数数量
# 最后一步：选择了某个完全平方数 j*j
# dp[n] = min(dp[n - j*j] + 1)  for all j*j <= n

import math

def num_squares(n):
    dp = [float('inf')] * (n + 1)
    dp[0] = 0  # base case
    
    for i in range(1, n + 1):
        j = 1
        while j * j <= i:
            dp[i] = min(dp[i], dp[i - j*j] + 1)
            j += 1
    
    return dp[n]

print(num_squares(12))  # 3 (4+4+4)
print(num_squares(13))  # 2 (4+9)
```

### 技巧3：前缀定义 vs 区间定义

**前缀定义**：`dp[i]` = 处理前 i 个元素的最优解（最常用）

**区间定义**：`dp[i][j]` = 处理区间 `[i, j]` 的最优解（区间DP用）

**示例**：两种方式解"最大子数组和"

```python
# 前缀定义（更常用，O(n)）
# dp[i] = 以 nums[i] 结尾的最大子数组和
def max_subarray_prefix(nums):
    dp = nums[:]
    for i in range(1, len(nums)):
        dp[i] = max(dp[i-1] + nums[i], nums[i])
    return max(dp)

# 区间定义（不推荐此题，O(n²)）
# dp[i][j] = nums[i..j] 的最大子数组和
# 定义复杂且效率低
```

### 技巧4：状态压缩

当原问题有 k 个布尔型子集状态时，用一个整数的二进制位来表示。

```python
# 问题：n 个任务，每个任务只能选或不选
# 枚举所有 2^n 种选法，用整数 mask 表示
for mask in range(1 << n):  # 0 到 2^n - 1
    for i in range(n):
        if mask & (1 << i):  # 第 i 个任务被选了
            ...
```

详见第12章（状压DP）。

### 技巧5：辅助状态

有时候，`dp[i]` 无法从同类型的前驱推导，需要**定义辅助状态**。

**示例**：最长摆动子序列（LeetCode 376）

```python
# dp[i][0] = 以 nums[i] 结尾，最后一步是"下降"的最长摆动子序列
# dp[i][1] = 以 nums[i] 结尾，最后一步是"上升"的最长摆动子序列

def wiggle_max_length(nums):
    n = len(nums)
    if n < 2:
        return n
    
    up = 1    # 最后一步是上升的摆动子序列长度
    down = 1  # 最后一步是下降的摆动子序列长度
    
    for i in range(1, n):
        if nums[i] > nums[i-1]:
            up = down + 1
        elif nums[i] < nums[i-1]:
            down = up + 1
    
    return max(up, down)
```

## 4.4 一道题，多种状态设计

**题目**：给定数组，每次可以删除连续相同元素，得分为删除数量的平方，求最大得分。

这是一道经典的难题（LeetCode 664 奇怪的打印机 相关），展示了状态设计的重要性：

**尝试1（失败）**：`dp[i]` = 处理前 i 个元素的最高分
- 问题：后续删除可能跨过中间元素与前面合并，违反无后效性

**尝试2（成功）**：`dp[i][j]` = 删除 `boxes[i..j]` 所有元素的最高分，并**附加**：在 i 前面还有 k 个与 `boxes[i]` 相同的元素可以一起删除
- 状态：`dp[i][j][k]`，三维DP

这说明：**有时需要"超前"设计状态，将隐含信息显式化**。

## 4.5 验证状态设计的方法

**检查清单**：

1. ✅ 状态定义是否精确（能否唯一确定子问题的答案）？
2. ✅ 满足无后效性（状态值不依赖历史路径）？
3. ✅ base case 是否覆盖所有边界情况？
4. ✅ 转移方程是否覆盖所有可能的来源？
5. ✅ 填表顺序是否满足依赖关系（被依赖的状态先计算）？

## 4.6 综合示例：买卖股票的最佳时机含冷冻期

**题目**（LeetCode 309）：
```
可以多次买卖，但卖出后需要冷冻1天才能再买入。
```

**状态设计**：
- `dp[i][0]`：第 i 天结束，**持有股票**，的最大利润
- `dp[i][1]`：第 i 天结束，**不持有股票且在冷冻期**，的最大利润
- `dp[i][2]`：第 i 天结束，**不持有股票且不在冷冻期**，的最大利润

**转移方程**：
```
dp[i][0] = max(dp[i-1][0],          # 继续持有
               dp[i-1][2] - prices[i])  # 从非冷冻期买入

dp[i][1] = dp[i-1][0] + prices[i]   # 今天卖出，进入冷冻期

dp[i][2] = max(dp[i-1][1],          # 昨天冷冻期结束
               dp[i-1][2])           # 昨天就不持有也不冷冻
```

```python
def max_profit_with_cooldown(prices):
    n = len(prices)
    if n <= 1:
        return 0
    
    dp = [[0] * 3 for _ in range(n)]
    dp[0][0] = -prices[0]  # 第0天买入
    dp[0][1] = 0
    dp[0][2] = 0
    
    for i in range(1, n):
        dp[i][0] = max(dp[i-1][0], dp[i-1][2] - prices[i])
        dp[i][1] = dp[i-1][0] + prices[i]
        dp[i][2] = max(dp[i-1][1], dp[i-1][2])
    
    return max(dp[n-1][1], dp[n-1][2])

print(max_profit_with_cooldown([1,2,3,0,2]))  # 3
```

## 4.7 本章小结

| 技巧 | 使用场景 |
|------|----------|
| 添加额外维度 | 状态信息不足，需要记录上下文 |
| 枚举最后一步 | 不知道如何推导转移方程 |
| 前缀 vs 区间定义 | 线性问题 vs 区间/合并问题 |
| 状态压缩 | 子集枚举，布尔型集合 |
| 辅助状态 | 一个维度无法区分不同情况 |

**DP 状态设计的黄金问题**：
> "在知道 `dp[i]` 的值后，我能正确地求出 `dp[i+1]` 吗？
> 如果不能，我还需要记录哪些额外信息？"
