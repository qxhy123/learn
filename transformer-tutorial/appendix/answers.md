## C.1 使用说明

### 如何使用本附录

本附录为全书24章共120道练习题提供答案索引、关键提示和参考代码要点。正确的使用方式是：

1. **先独立完成练习**：在查阅答案之前，请务必自己尝试解答。对于代码题，建议至少写出完整的思路框架，并运行验证。
2. **遇到困难时查看提示**：每道题都提供了关键知识点提示，这些提示通常足以帮助你找到解题方向，而不必直接看完整答案。
3. **答案在各章末尾**：部分章节（如第13、14、15、19、21、24章）已在章节末尾附有完整答案。本附录提供的是跨章节的快速索引和补充提示，而非重复完整答案。
4. **代码题重在过程**：代码实现题不存在唯一正确答案，本附录给出的是参考实现要点和容易出错的位置，鼓励读者探索不同实现方式。

### 难度说明

每章5道题按以下难度分级：

| 难度 | 数量 | 特征 |
|------|------|------|
| 基础题 | 2道 | 概念理解、手算验证、参数量计算 |

基础题通常可在30分钟内完成；中级题需要1-3小时；提高题可能需要半天到一天。

### 第1章：序列建模基础

| 题号 | 题目描述 | 难度 | 关键知识点 |
|------|----------|------|------------|
| 1.1 | RNN特性多选题 | 基础 | 权重共享、并行化限制、梯度消失 |
| 1.2 | LSTM参数量计算 | 基础 | 四个门的参数公式、双向双层扩展 |
| 1.3 | 双向LSTM情感分类器 | 中级 | pack_padded_sequence、双向拼接 |
| 1.4 | 梯度消失可视化实验 | 中级 | 反向传播、梯度范数随时间步衰减 |
| 1.5 | 带注意力的Seq2Seq | 提高 | Bahdanau注意力、对齐矩阵可视化 |

**答题要点**：

- **1.1**：正确选项为 A、B、E。C错误，RNN时间步之间存在数据依赖，无法并行；D错误，隐藏维度与输入维度相互独立。
- **1.2**：单层LSTM参数量 = $4 \times (d_x \times d_h + d_h \times d_h + d_h)$。代入 $d_x=128, d_h=256$：
  - 单层单向 = $4 \times (128 \times 256 + 256 \times 256 + 256) = 4 \times (32768 + 65536 + 256) = 394240$
  - 双层双向：第一层双向参数量需注意第二层输入维度变为 $2 \times d_h = 512$
  - LSTM参数量约为同规模RNN的4倍（因为有4个门控）
- **1.3**：关键是使用 `pack_padded_sequence` 处理变长序列，前向后向隐状态拼接时注意维度顺序（`h[-2,:,:]` 为前向最后状态，`h[-1,:,:]` 为后向）
- **1.4**：用 `register_hook` 在每个时间步记录梯度范数，绘制对数坐标图能清晰看到指数衰减

### 第2章：注意力机制原理

| 题号 | 题目描述 | 难度 | 关键知识点 |
|------|----------|------|------------|
| 2.1 | 手算点积注意力 | 基础 | 矩阵乘法、softmax数值计算 |
| 2.2 | 注意力机制特性多选 | 基础 | 缩放因子方差分析、因果掩码 |
| 2.3 | 缩放的数值效应分析 | 中级 | 方差推导、softmax梯度饱和 |
| 2.4 | 实现加性注意力并对比 | 中级 | Bahdanau加性注意力、可学习参数 |
| 2.5 | 注意力机制综合应用 | 提高 | 多种注意力变体对比实验 |

**答题要点**：

- **2.1**：
  - $S = QK^T$ 形状为 $(2, 3)$，$S_{11} = 1\times1 + 2\times0 = 1$，$S_{12} = 1\times0 + 2\times1 = 2$，$S_{13} = 1\times1 + 2\times1 = 3$
  - 第一行 softmax：$\exp([1,2,3]) = [e, e^2, e^3]$，归一化后约为 $[0.090, 0.245, 0.665]$
  - 输出向量 = 权重 × V 的加权和
- **2.2**：正确选项为 B、C、E。A错误，缩放因子将方差从 $d_k$ 缩放到1（不是均值）；D错误，Bahdanau注意力包含可学习的对齐权重矩阵
- **2.3**：
  - $q \cdot k$ 的期望为0，方差为 $d_k$（各分量独立，方差累加）
  - 缩放后方差为1
  - 不缩放时 $\exp(32)$ 极大，softmax输出接近 one-hot，梯度趋近于0

| 3.2 | 可学习位置编码参数量 | 基础 | BERT位置嵌入大小、外推局限 |

### 第4章：自注意力机制

| 题号 | 题目描述 | 难度 | 关键知识点 |
|------|----------|------|------------|
| 4.1 | 张量形状推导 | 基础 | Q/K/V投影、矩阵乘法输出形状 |
| 4.2 | 缩放因子的必要性 | 基础 | $\sqrt{d_k}$ vs $d_k$ vs $\sqrt{d_{model}}$，softmax梯度 |
| 4.3 | 添加输出投影层 | 中级 | $W^O$ 的作用、维度恢复 |
| 4.4 | 因果掩码自注意力 | 中级 | 下三角掩码生成、`-inf` 填充 |
| 4.5 | 注意力可视化实验 | 提高 | BertViz风格热图、不同层的注意力模式 |

**答题要点**：

- **4.1**：中间张量形状追踪：
  - $Q = XW^Q$：$(3,4) \times (4,2) \to (3,2)$
  - $K = XW^K$：$(3,2)$
  - $S = QK^T$：$(3,2) \times (2,3) \to (3,3)$
  - $A = \text{softmax}(S)$：$(3,3)$
  - $V = XW^V$：$(3,4)$
  - 输出 $= AV$：$(3,3) \times (3,4) \to (3,4)$
- **4.2**：除以 $\sqrt{d_k}$ 使得点积方差保持在1附近；若不缩放，大维度下点积值很大，softmax输出趋近 one-hot，梯度消失；应除以 $\sqrt{d_k}$（不是 $d_k$ 也不是 $\sqrt{d_{model}}$，因为Q和K的维度是 $d_k$）
- **4.4**：使用 `torch.triu(torch.ones(L, L), diagonal=1).bool()` 生成上三角掩码，在得分矩阵对应 `True` 的位置填 `float('-inf')`，确保 softmax 后这些位置权重为0

### 第7章：编码器结构

| 题号 | 题目描述 | 难度 | 关键知识点 |
|------|----------|------|------------|
| 7.1 | 手算LayerNorm | 基础 | 均值、方差、归一化公式 |
| 7.2 | 编码器层参数量分析 | 基础 | 注意力部分 vs FFN部分比例 |
| 7.3 | 将激活函数改为SwiGLU | 中级 | $\text{SwiGLU}(x) = \text{Swish}(xW_1) \odot (xW_2)$，参数量保持 |
| 7.4 | Pre-LN vs Post-LN稳定性实验 | 中级 | 梯度尺度分析、初始训练损失曲线 |
| 7.5 | 稀疏MoE FFN实现 | 提高 | Top-K路由、负载均衡损失 |

**答题要点**：

- **7.1**：$\mu = (1+2+3+4)/4 = 2.5$，$\sigma^2 = (2.25+0.25+0.25+2.25)/4 = 1.25$，$\hat{x} = \frac{x-\mu}{\sqrt{\sigma^2+\epsilon}}$，代入 $\gamma=\mathbf{1}, \beta=\mathbf{0}$ 后输出 $\approx [-1.342, -0.447, 0.447, 1.342]$
- **7.2**：注意力部分参数 = $4 \times d_{model}^2 = 4 \times 512^2 = 1M$；FFN部分 = $2 \times d_{model} \times d_{ff} = 2 \times 512 \times 2048 = 2M$；FFN占约67%
- **7.3**：SwiGLU需要两个线性层（维度各为 $d_{model} \to d_{ff}$），取 $d_{ff} = \lfloor 8d_{model}/3 \rfloor$ 使参数量与标准FFN（$d_{ff}=4d_{model}$，单线性层）相当
- **7.5**：辅助损失 $\mathcal{L}_{aux} = E \cdot \sum_i f_i \cdot P_i$，其中 $f_i$ 是路由到专家 $i$ 的token比例（不可微），$P_i$ 是路由softmax输出的均值（可微）；两者乘积使梯度能传播

### 第10章：训练策略

> **注意**：本章文件尚未创建，以下为根据课程大纲推断的练习题方向。

### 第11章：优化技术

| 题号 | 题目描述 | 难度 | 关键知识点 |
|------|----------|------|------------|
| 11.1 | Adam L2正则化的问题 | 基础 | Adam与AdamW的区别、权重衰减解耦 |
| 11.2 | 梯度累积步数计算 | 基础 | 等效批次大小 = 实际批次 × 累积步数 × GPU数 |
| 11.3 | 参数分组（含/不含权重衰减） | 中级 | Embedding/LayerNorm/bias不衰减 |
| 11.4 | 混合精度训练实现 | 中级 | `torch.cuda.amp`、GradScaler |
| 11.5 | 分布式训练数据并行 | 提高 | DDP、DistributedSampler、梯度同步 |

**答题要点**：

- **11.1**：问题在于 `Adam + weight_decay` 将 L2 惩罚耦合到自适应学习率中，等效衰减量因参数的梯度历史而异；AdamW 将权重衰减解耦，直接对参数值施加固定比例衰减：`param.data *= (1 - lr * weight_decay)`
- **11.2**：`accumulation_steps = 目标批次 / (实际批次 × GPU数) = 256 / (8 × 4) = 8`；损失应除以 `accumulation_steps`（在 `loss.backward()` 之前）以使每步梯度等效
- **11.3**：分组逻辑：遍历所有命名参数，根据名称中是否含 `layer_norm`/`layernorm`/`bias`/`embed_tokens` 决定放入无衰减组还是衰减组；传入 `optimizer` 的 `param_groups` 列表

### 第12章：损失函数与评估

| 题号 | 题目描述 | 难度 | 关键知识点 |
|------|----------|------|------------|
| 12.1 | 手算交叉熵与困惑度 | 基础 | 忽略PAD、对数计算、$\text{PPL} = e^{CE}$ |
| 12.2 | reduction参数对比分析 | 基础 | `mean` vs `sum`、多GPU梯度聚合 |
| 12.3 | 手动实现语料库级BLEU | 中级 | N-gram精度、简短惩罚BP |
| 12.4 | ROUGE实现与对比 | 中级 | ROUGE-1/2/L，召回率 vs F1 |
| 12.5 | 评估指标综合分析 | 提高 | BLEU/ROUGE/BERTScore适用场景 |

**答题要点**：

- **12.1**：有效位置3个（忽略PAD）；$CE = -((\log 0.6 + \log 0.8 + \log 0.4)/3) = -(-0.511-0.223-0.916)/3 \approx 0.55$；$\text{PPL} = e^{0.55} \approx 1.73$
- **12.2**：`loss_A = loss_B / 有效token数`；多GPU分布式训练中应使用 `reduction='sum'` 再手动除以全局有效token总数，避免不同设备上有效token数不一致导致的梯度偏差

## C.3 常见错误提示

### 代码实现的常见 Bug

**1. 维度顺序问题**

多头注意力中最常见的错误是 `view` 和 `transpose` 的顺序。正确顺序是：
```python
# 正确：先 view 分离头维度，再 transpose 调换头与序列维度
x = x.view(B, L, h, d_k).transpose(1, 2)  # → (B, h, L, d_k)

# 错误：直接 view 成 (B, h, L, d_k) 会导致头间数据混淆
x = x.view(B, h, L, d_k)  # 数据排列错误！
```

**2. 掩码填充值错误**

因果掩码应填 `float('-inf')` 而非 `-1e9`（虽然后者在大多数情况下有效，但在数值边界情况下可能出现问题）：
```python
# 推荐
scores = scores.masked_fill(mask, float('-inf'))
# 不推荐（有时会导致softmax后出现NaN）
scores = scores.masked_fill(mask, -1e9)
```

**3. 梯度累积时损失归一化遗忘**

```python
# 正确：损失除以累积步数
loss = loss / accumulation_steps
loss.backward()

# 错误：直接反向传播，等效批次的梯度被放大
loss.backward()
```

**4. LayerNorm vs BatchNorm 的维度参数**

```python
# LayerNorm：normalized_shape 是最后几个维度
nn.LayerNorm(d_model)              # 正确，归一化最后一维

# 常见错误：传入完整形状
nn.LayerNorm([batch_size, seq_len, d_model])  # 错误，batch_size在推理时变化
```

**5. LoRA 合并时的权重顺序**

```python
# 正确：W_new = W_frozen + (B @ A) * scaling
merged = self.weight + (self.lora_B @ self.lora_A) * (self.alpha / self.r)

# 错误：A @ B 矩阵乘法顺序反了
merged = self.weight + (self.lora_A @ self.lora_B) * (self.alpha / self.r)
```

### 概念理解的常见误区

**误区1：自注意力一定优于RNN**

自注意力的计算复杂度为 $O(L^2 d)$，在序列极长时（如 $L > 10^4$）反而不如 RNN 高效。高效Transformer（Longformer、Linformer等）正是为此而生。

**误区2：更多注意力头总是更好**

注意力头数增加并不增加参数量（总参数 = $4d_{model}^2$，与头数无关），但每头的表达维度 $d_k = d_{model}/h$ 变小。头数过多时每头表达能力受限，实际性能可能下降。通常 $d_k \geq 32$ 是经验下限。

**误区3：Pre-LN总是优于Post-LN**

Pre-LN（层归一化在残差前）训练更稳定，但部分研究表明Post-LN在充分调参后性能略优。两者各有适用场景，原始论文使用Post-LN，现代大模型普遍采用Pre-LN。

**误区4：困惑度越低模型一定越好**

困惑度是在特定测试集上的指标，不同分词方式会导致无法跨模型直接比较。此外，困惑度低不代表生成质量高（可能倾向于产出高频平庸文本）。

**误区5：LoRA rank越大效果越好**

过大的 rank 会引入过多可训练参数，在小数据集上可能导致过拟合，同时失去PEFT的参数效率优势。通常 $r \in \{4, 8, 16\}$ 是实践中的合理范围。

### 调试技巧

**1. 梯度检查（Gradient Check）**

怀疑反向传播实现有误时，用数值梯度验证：
```python
from torch.autograd import gradcheck
# 注意输入需要 double 精度，eps 适当放大
result = gradcheck(my_function, inputs, eps=1e-4, atol=1e-3)
```

**2. 注意力权重检查**

训练初期注意力权重应接近均匀分布（熵接近 $\log L$）；若一开始就极端集中，检查是否忘记除以 $\sqrt{d_k}$ 或掩码填充值过小。

**3. 损失不下降的排查顺序**

① 检查学习率（过大/过小）→ ② 检查损失计算（是否忘记 `ignore_index`）→ ③ 检查数据预处理（标签偏移是否正确）→ ④ 检查梯度流（是否存在梯度截断或消失）→ ⑤ 用极小数据集过拟合验证模型容量

**4. 显存溢出（OOM）排查**

按以下顺序缩减：① 减小批次大小 → ② 启用梯度检查点 → ③ 使用混合精度（FP16）→ ④ 减少序列长度 → ⑤ 减小模型维度

**5. 多GPU训练不一致**

DDP训练时若不同rank的损失不一致，检查：① `DistributedSampler` 是否在每个epoch重新设置 seed → ② 是否正确使用 `loss.backward()` 前同步梯度 → ③ BatchNorm是否替换为 `SyncBatchNorm`（Transformer通常用LayerNorm，无此问题）
