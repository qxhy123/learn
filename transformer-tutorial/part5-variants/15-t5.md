## 引言

2020年，Google Research发布了一篇影响深远的论文——《Exploring the Limits of Transfer Learning with a Unified Text-to-Text Transformer》（T5）。这篇论文不仅提出了一个强大的预训练模型，更重要的是提出了一种全新的思维框架：**将所有NLP任务统一为文本到文本（Text-to-Text）的生成问题**。

在T5之前，NLP领域存在一种隐性的"任务特化"思维：分类任务要加分类头，序列标注任务要加CRF层，问答任务要预测span的起止位置。每种任务都需要不同的输出头和损失函数，这导致模型结构的碎片化。

T5的核心洞见是：**所有这些任务都可以转化为"输入一段文本，输出一段文本"的形式**。翻译是文本到文本，摘要是文本到文本，分类？也是文本到文本——只不过输出是"正面"或"负面"这样的词而已。

这种统一框架带来的好处是深刻的：同一套模型架构、同一套损失函数、同一套解码策略，可以处理几乎所有NLP任务。多任务学习也因此变得异常自然。

## 15.1 Text-to-Text框架

### 15.1.1 统一的输入输出格式

Text-to-Text框架的核心思想非常简洁：**无论什么任务，都将其格式化为"输入文本 → 输出文本"的形式**。

```
输入文本  ──→  T5模型  ──→  输出文本
```

这与以往的范式形成了鲜明对比：

| 任务类型 | 传统范式 | Text-to-Text范式 |
|----------|----------|-----------------|
| 文本分类 | 输出类别概率向量 | 输出类别名称字符串 |
| 机器翻译 | seq2seq输出目标语言 | 输出目标语言文本 |
| 问答 | 预测答案的起止位置 | 输出答案字符串 |
| 文本摘要 | seq2seq输出摘要 | 输出摘要文本 |
| 自然语言推理 | 输出蕴含/矛盾/中性的概率 | 输出"entailment"/"contradiction"/"neutral" |

### 15.1.2 任务前缀（Task Prefix）

既然所有任务的输入输出格式都相同，模型如何知道当前要执行什么任务？答案是**任务前缀（Task Prefix）**。

在每个输入文本前面加上一个描述任务的前缀，告知模型应该执行什么操作：

```
翻译任务：
输入：  "translate English to German: The house is wonderful."
输出：  "Das Haus ist wunderbar."

摘要任务：
输入：  "summarize: state authorities dispatched emergency response teams..."
输出：  "state authorities sent emergency teams."

情感分类：
输入：  "sst2 sentence: The movie is great!"
输出：  "positive"

自然语言推理：
输入：  "mnli premise: I hate pigeons. hypothesis: My feelings towards pigeons are filled with animosity."
输出：  "entailment"
```

任务前缀的设计哲学非常实用：它不依赖模型的特殊结构，仅通过文本本身传达任务信息。这使得T5可以在推理时通过简单修改前缀来切换任务。

### 15.1.3 从所有任务中学习

Text-to-Text框架使得**多任务联合训练**变得极为自然。可以将来自不同任务的数据混合在一起，共同训练一个模型。每条训练样本都附带任务前缀，模型在训练过程中同时学习如何执行多种任务。

这与人类学习的方式更接近：我们在学习阅读、写作、翻译的过程中，知识是相互迁移的，而不是各自独立的。

### 15.1.4 示例：四种任务的统一表示

下面用一个具体例子，展示四种不同任务在Text-to-Text框架下如何统一表示：

**翻译（英译法）：**
```
输入：  "translate English to French: That is good."
输出：  "C'est bon."
```

**摘要：**
```
输入：  "summarize: The US has passed the law that makes it illegal to ..."
输出：  "The law makes it illegal to ..."
```

**文本分类（CoLA语法可接受性）：**
```
输入：  "cola sentence: The course is jumping well."
输出：  "unacceptable"
```

**回归任务（STS-B语义相似度，输出数字）：**
```
输入：  "stsb sentence1: The rhino grazed on the grass. sentence2: A rhino is grazing in a field."
输出：  "3.8"
```

注意最后一个例子：即使是回归任务（输出连续数值），T5也将其转化为生成一个数字字符串的文本任务。这是Text-to-Text框架灵活性的极致体现。

## 15.2 T5架构

### 15.2.1 编码器-解码器结构

T5采用标准的**编码器-解码器（Encoder-Decoder）**架构，这与原始Transformer一致。这个选择并非偶然——对于Text-to-Text任务，编码器-解码器架构天然匹配：编码器处理输入文本，解码器自回归地生成输出文本。

```
        编码器侧                         解码器侧
   ┌─────────────────┐             ┌─────────────────┐
   │  输入Token嵌入  │             │  输出Token嵌入  │
   └────────┬────────┘             └────────┬────────┘
            │                               │
   ┌────────▼────────┐             ┌────────▼────────┐
   │  自注意力层     │             │  掩码自注意力层  │
   │  (全连接)       │             │  (因果掩码)      │
   └────────┬────────┘             └────────┬────────┘
            │                               │
   ┌────────▼────────┐             ┌────────▼────────┐
   │  前馈网络层     │             │  交叉注意力层   │◄── 编码器输出
   └────────┬────────┘             └────────┬────────┘
            │  (×N)                         │
            │                      ┌────────▼────────┐
            │                      │  前馈网络层     │
            │                      └────────┬────────┘
            │                               │  (×N)
            ▼                               ▼
        编码器输出                      线性+Softmax
```

T5的编码器和解码器各有12层（Base版本），每层包含自注意力、（解码器还有交叉注意力）和前馈网络。

### 15.2.2 相对位置偏置（Relative Position Bias）

原始Transformer使用绝对位置编码（正弦函数或可学习嵌入）。T5引入了一种更简洁的**相对位置偏置**方案。

**绝对位置编码的问题：**
- 无法外推到训练时未见过的序列长度
- 位置信息与token内容混合在同一个嵌入向量中

**T5的相对位置偏置：**

在计算注意力分数时，直接加上一个基于相对位置的偏置值：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} + b_{i-j}\right) V$$

其中 $b_{i-j}$ 是第 $i$ 个位置关注第 $j$ 个位置时的偏置值，只依赖相对距离 $i-j$。

具体实现中，T5对相对位置进行了**分桶（bucketing）**处理：

```python
def relative_position_bucket(relative_position, num_buckets=32, max_distance=128):
    """
    将相对位置映射到桶索引
    - 小距离：精细划分（线性）
    - 大距离：粗略划分（对数）
    """
    ret = 0
    n = -relative_position

    # 前半部分桶用于负相对位置（即关注左边的位置）
    num_buckets //= 2
    ret += (n < 0).to(torch.long) * num_buckets
    n = torch.abs(n)

    # 小距离线性划分，大距离对数划分
    max_exact = num_buckets // 2
    is_small = n < max_exact

    val_if_large = max_exact + (
        torch.log(n.float() / max_exact) /
        math.log(max_distance / max_exact) *
        (num_buckets - max_exact)
    ).to(torch.long)
    val_if_large = torch.min(val_if_large,
                             torch.full_like(val_if_large, num_buckets - 1))

    ret += torch.where(is_small, n, val_if_large)
    return ret
```

这种设计的优点是：
1. 相对位置信息直接编码在注意力计算中，不占用嵌入维度
2. 对数分桶使模型能够泛化到更长的序列
3. 跨层**共享**位置偏置参数，减少参数量

### 15.2.3 简化的层归一化（RMSNorm）

T5对原始的层归一化（LayerNorm）进行了简化，去掉了减均值的中心化操作，只保留均方根归一化：

**原始LayerNorm：**
$$\text{LayerNorm}(x) = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \cdot \gamma + \beta$$

**T5的RMSNorm（Root Mean Square Normalization）：**
$$\text{RMSNorm}(x) = \frac{x}{\text{RMS}(x)} \cdot \gamma, \quad \text{RMS}(x) = \sqrt{\frac{1}{n}\sum_{i=1}^{n} x_i^2}$$

RMSNorm去掉了均值计算和偏置参数 $\beta$，更加简洁高效，实践中效果相当。

此外，T5将归一化放在每个子层**之前**（Pre-Norm），而不是原始Transformer的之后（Post-Norm）。Pre-Norm在训练深层网络时梯度更稳定。

### 15.2.4 与原始Transformer的主要区别

| 特性 | 原始Transformer | T5 |
|------|----------------|-----|
| 位置编码 | 正弦绝对位置编码 | 相对位置偏置 |
| 层归一化 | Post-Norm, LayerNorm | Pre-Norm, RMSNorm |
| 偏置项 | 有 | 全部去掉 |
| Dropout | 有 | 有（率更低） |
| 激活函数 | ReLU | GELU（后续版本改为GeGLU） |
| 词嵌入共享 | 不共享 | 编码器/解码器/输出层三方共享 |

**词嵌入三方共享**是T5的重要设计：编码器输入嵌入、解码器输入嵌入、解码器输出的线性投影层，三者共享同一套参数矩阵。这大幅减少了参数量，同时让输入输出空间保持一致。

## 15.3 预训练任务

### 15.3.1 Span Corruption任务

T5的预训练任务称为**Span Corruption**（片段损坏），是BERT的MLM任务的改进版本。

**BERT的MLM任务：**
- 随机掩盖15%的**单个token**
- 模型预测被掩盖的token
- 输出与输入等长

**T5的Span Corruption：**
- 随机选择若干**连续片段（span）**进行掩盖
- 每个片段用一个**哨兵token（Sentinel Token）**替换
- 模型需要生成所有被掩盖的片段内容

```
原始文本：
"Thank you for inviting me to your party last week."

输入（掩盖后）：
"Thank you <extra_id_0> me to your party <extra_id_1> week."

目标输出：
"<extra_id_0> for inviting <extra_id_1> last <extra_id_2>"
```

默认参数：掩盖15%的token，平均span长度为3。

### 15.3.2 Sentinel Token（哨兵Token）

哨兵Token是T5预训练的关键设计。每个被掩盖的片段用一个唯一的特殊token表示：`<extra_id_0>`, `<extra_id_1>`, ..., `<extra_id_99>`。

这些哨兵token有两个作用：
1. **输入端**：标记被掩盖片段的位置和数量
2. **输出端**：作为分隔符，将不同片段的还原内容区分开

目标序列的结构如下：
$$\text{target} = [\text{sentinel}_0, \text{span}_0, \text{sentinel}_1, \text{span}_1, \ldots, \text{sentinel}_{k-1}, \text{span}_{k-1}, \text{sentinel}_k]$$

最后一个哨兵token表示序列结束（或者用EOS token）。

### 15.3.3 不同掩码策略的对比

T5论文对多种预训练目标进行了系统比较：

| 预训练目标 | 描述 | GLUE平均分 | 相对速度 |
|-----------|------|-----------|---------|
| 语言模型（GPT风格） | 自回归预测下一个token | 基线 | 慢 |
| BERT风格MLM | 掩盖单个token，双向预测 | +2.3 | 慢 |
| Deshuffling | 重排序打乱的句子 | -1.1 | 慢 |
| MASS风格 | 掩盖并生成连续片段 | +1.8 | 中 |
| **Span Corruption（T5默认）** | **掩盖并生成多个片段** | **+2.5** | **快** |
| Prefix LM | 前缀双向，后缀单向 | +1.9 | 中 |

Span Corruption的优势在于：
- 模型需要学习理解上下文（因为片段可能跨越多个token）
- 目标序列比原始序列短很多，训练效率高
- 既学到双向理解（编码器），又学到生成能力（解码器）

### 15.3.4 预训练数据（C4数据集）

T5的预训练数据集称为**C4（Colossal Clean Crawled Corpus）**，是从CommonCrawl网络爬取数据中经过严格清洗后得到的语料。

清洗流程：
1. 只保留以句号、感叹号、问号结尾的行（过滤代码等非自然语言）
2. 去掉包含不良词汇的页面
3. 去掉包含代码标记（`{`）的行
4. 去掉少于3个句子的页面
5. 去重（3-gram指纹）
6. 只保留英文内容（langdetect检测）

最终C4包含约**750GB**的纯文本数据，约**156亿个token**，是当时最大的预训练语料之一。

## 15.4 多任务学习

### 15.4.1 混合多个任务训练

T5的多任务学习通过将来自不同任务的数据混合在同一个训练批次中实现。每条数据都附带任务前缀，模型通过前缀区分不同任务：

```
批次示例（混合多任务）：
[sample_1]  "translate English to German: Hello world."  → "Hallo Welt."
[sample_2]  "summarize: The government announced..."      → "Government plans..."
[sample_3]  "sst2 sentence: Great movie!"               → "positive"
[sample_4]  "mnli premise: ... hypothesis: ..."         → "entailment"
[sample_5]  "cola sentence: The dogs barks."            → "unacceptable"
```

所有样本使用相同的模型参数、相同的交叉熵损失函数、相同的优化器更新规则。不同任务之间自然地共享知识。

### 15.4.2 任务采样策略

当多个任务的数据量差异较大时，如何采样是关键问题。T5论文比较了三种策略：

**1. 均匀采样（Examples-proportional mixing）：**
按照每个任务的数据量比例采样。数据量大的任务主导训练，小任务容易欠拟合。

**2. 温度采样（Temperature-scaled mixing）：**
$$p_i \propto \left(\frac{n_i}{\sum_j n_j}\right)^{1/T}$$

温度 $T$ 控制均匀程度：
- $T=1$：按数据量比例采样
- $T \to \infty$：均匀采样（每个任务等概率）
- T5默认使用 $T=2$，在数据量大的任务和小任务之间取得平衡

**3. 等比例采样（Equal mixing）：**
每个任务等概率采样，但数据少的任务会频繁重复。

实验表明温度采样（T=2）效果最好，它在不过度忽略小任务的同时，也不让大任务的优势完全消失。

### 15.4.3 任务间的知识迁移

多任务学习的核心价值在于**正向迁移（Positive Transfer）**：在一个任务上学到的知识能帮助提升其他任务的性能。

T5论文的实验揭示了一些有趣的迁移模式：
- 翻译任务能帮助提升摘要任务（都需要理解语义并生成流畅文本）
- 自然语言推理任务能帮助提升问答任务（都需要推理能力）
- 摘要任务的训练对翻译任务有轻微负迁移（可能因为摘要允许改写，而翻译需要忠实）

**预训练 → 多任务微调 → 单任务微调** 的三阶段流程往往比直接单任务微调效果更好：

```
阶段1（预训练）：  C4上的Span Corruption（无监督）
        ↓
阶段2（多任务）：  混合所有下游任务数据（有监督）
        ↓
阶段3（单任务）：  在目标任务上精细调整（有监督）
```

### 15.4.4 提示工程（Prompt Engineering）

T5的任务前缀是早期**提示工程（Prompt Engineering）**的雏形。通过精心设计输入文本的格式，可以引导模型执行特定操作，而无需修改模型参数。

在T5中，不同的任务前缀格式会显著影响性能：

```python
# 效果较好的前缀
"translate English to German: {text}"     # 明确的语言对
"summarize: {text}"                        # 简洁
"question: {q} context: {ctx}"            # 明确的结构

# 效果较差的前缀
"EN to DE: {text}"                         # 缩写不明确
"please summarize the following: {text}"   # 冗余
```

T5的这些发现直接启发了后续GPT-3的In-Context Learning和Prompt Tuning等技术，标志着NLP从"任务特化模型"向"通用提示驱动模型"转变的开端。

## 15.5 T5变体

### 15.5.1 T5模型系列

T5按照参数量划分为多个规模：

| 模型 | 参数量 | 编码器层数 | 解码器层数 | 隐藏维度 | 注意力头数 |
|------|--------|-----------|-----------|---------|----------|
| T5-Small | 60M | 6 | 6 | 512 | 8 |
| T5-Base | 220M | 12 | 12 | 768 | 12 |
| T5-Large | 770M | 24 | 24 | 1024 | 16 |
| T5-3B | 3B | 24 | 24 | 1024 | 32 |
| T5-11B | 11B | 24 | 24 | 1024 | 128 |

T5-11B在发布时在多项NLP基准测试上达到了SOTA水平，展示了规模扩展的强大效果。

### 15.5.2 mT5（多语言T5）

**mT5（Multilingual T5）** 是T5的多语言版本，支持101种语言。

主要改动：
- 预训练数据：从mC4（多语言C4，网页爬取的101种语言数据）预训练
- 词表：扩展到250,000个token（原T5为32,000），以覆盖多语言字符
- 架构：与T5相同，未做结构修改

mT5的一个重要发现是**跨语言零样本迁移**：在英文任务上微调后，模型可以直接处理其他语言的相同任务，无需该语言的标注数据。这表明多语言预训练学到了深层的语言无关表示。

### 15.5.3 FLAN-T5（指令微调版本）

**FLAN-T5（Fine-tuned Language Net T5）** 是经过大规模指令微调的T5版本。

**指令微调（Instruction Fine-tuning）的核心思想：**
用大量包含自然语言指令的任务数据对模型进行微调，使模型更好地理解和遵循人类指令。

FLAN-T5的训练数据覆盖1836个任务，包括：
- 翻译（数百个语言对）
- 摘要（多种风格）
- 问答（多种格式）
- 数学推理
- 代码生成
- 常识推理
- 对话

关键发现：**指令微调的效果随模型规模增大而显著增强**。FLAN-T5-XXL（11B）在zero-shot和few-shot设置下大幅超越原始T5，甚至在某些任务上超越GPT-3（175B）。

```
FLAN-T5在MMLU（多学科知识测试）上的zero-shot得分：
FLAN-T5-Small  (60M):   ~45%
FLAN-T5-Base   (250M):  ~52%
FLAN-T5-Large  (780M):  ~56%
FLAN-T5-XL     (3B):    ~60%
FLAN-T5-XXL    (11B):   ~65%
GPT-3          (175B):  ~43%  ← 规模大但无指令微调
```

### 15.5.4 UL2（统一语言学习）

**UL2（Unifying Language Learning Paradigms）** 进一步发展了T5的统一框架思想，提出用一个模型同时学习多种语言建模范式。

UL2定义了三类"去噪器（Denoiser）"：

| 去噪器类型 | 描述 | 前缀 | 适合的能力 |
|-----------|------|------|-----------|
| R-Denoiser | 短span损坏（类似T5的span corruption） | `[R]` | 事实知识、NLU |
| S-Denoiser | 长span损坏（后缀预测） | `[S]` | 长文本生成 |
| X-Denoiser | 极端损坏（掌握50%token） | `[X]` | 复杂推理 |

在推理时，通过在输入前加对应前缀（`[R]`, `[S]`, `[X]`），可以激活模型不同的"工作模式"。这使得UL2兼具NLU和NLG的强大能力。

UL2-20B在SuperGLUE等基准上超越了GPT-3（175B），再次证明训练策略比单纯堆参数更重要。

---

## 本章小结

本章我们深入学习了T5及其统一框架。T5最重要的贡献不仅是一个具体的模型，更是一种**思维范式的转变**：将NLP的所有任务统一为文本到文本的生成问题。

下表对T5、BERT和GPT三类代表性架构进行系统对比：

| 特性 | BERT | GPT | T5 |
|------|------|-----|-----|
| **架构类型** | 仅编码器 | 仅解码器 | 编码器-解码器 |
| **预训练任务** | MLM + NSP | 下一个token预测 | Span Corruption |
| **注意力方向** | 双向（全连接） | 单向（因果掩码） | 编码器双向+解码器单向 |
| **任务适配方式** | 添加任务特定头 | 提示/生成 | 文本到文本（任务前缀） |
| **生成能力** | 弱（非自回归设计） | 强 | 强 |
| **理解能力** | 强 | 中 | 强 |
| **位置编码** | 绝对位置嵌入 | 绝对位置嵌入 | 相对位置偏置 |
| **归一化** | Post-Norm LayerNorm | Post-Norm LayerNorm | Pre-Norm RMSNorm |
| **代表模型** | BERT-large (340M) | GPT-3 (175B) | T5-11B |
| **典型应用** | 分类、NER、QA | 文本生成、对话 | 所有NLP任务 |

T5之后，编码器-解码器架构的统一框架成为NLP模型设计的重要方向。其核心理念——用文本格式统一所有任务——直接影响了后续的提示学习、指令微调等技术，并在当今大语言模型（LLM）的设计中得到充分体现。

### 15.6.1 T5模型架构概览

下面用简化代码展示T5的核心组件：

```python
import torch
import torch.nn as nn
import math


class T5RMSNorm(nn.Module):
    """T5使用的RMSNorm（去掉了均值中心化）"""

    def __init__(self, hidden_size: int, eps: float = 1e-6):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(hidden_size))
        self.eps = eps

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # 计算均方根
        rms = x.pow(2).mean(-1, keepdim=True).add(self.eps).sqrt()
        return self.weight * (x / rms)


class T5RelativePositionBias(nn.Module):
    """T5的相对位置偏置"""

    def __init__(self, num_heads: int, num_buckets: int = 32, max_distance: int = 128):
        super().__init__()
        self.num_heads = num_heads
        self.num_buckets = num_buckets
        self.max_distance = max_distance
        # 每个注意力头有独立的位置偏置
        self.relative_attention_bias = nn.Embedding(num_buckets, num_heads)

    @staticmethod
    def _relative_position_bucket(relative_position, num_buckets=32, max_distance=128):
        """将相对位置映射到桶索引"""
        ret = 0
        n = -relative_position  # 使负数变正

        # 一半桶用于正方向，一半用于负方向
        num_buckets //= 2
        ret += (n < 0).long() * num_buckets
        n = torch.abs(n)

        # 小距离线性，大距离对数
        max_exact = num_buckets // 2
        is_small = n < max_exact

        val_if_large = max_exact + (
            torch.log(n.float() / max_exact) /
            math.log(max_distance / max_exact) *
            (num_buckets - max_exact)
        ).long()
        val_if_large = torch.clamp(val_if_large, max=num_buckets - 1)

        ret += torch.where(is_small, n, val_if_large)
        return ret

    def forward(self, query_length: int, key_length: int) -> torch.Tensor:
        """
        返回形状: (1, num_heads, query_length, key_length)
        """
        device = self.relative_attention_bias.weight.device

        # 构建相对位置矩阵
        query_pos = torch.arange(query_length, device=device).unsqueeze(1)  # (q, 1)
        key_pos = torch.arange(key_length, device=device).unsqueeze(0)      # (1, k)
        relative_position = key_pos - query_pos  # (q, k)

        # 映射到桶索引
        rp_bucket = self._relative_position_bucket(
            relative_position,
            num_buckets=self.num_buckets,
            max_distance=self.max_distance
        )

        # 查表得到偏置值
        values = self.relative_attention_bias(rp_bucket)  # (q, k, num_heads)
        values = values.permute(2, 0, 1).unsqueeze(0)     # (1, num_heads, q, k)
        return values


class T5Attention(nn.Module):
    """T5注意力层（含相对位置偏置）"""

    def __init__(self, hidden_size: int, num_heads: int,
                 is_cross_attention: bool = False,
                 has_relative_bias: bool = True):
        super().__init__()
        self.hidden_size = hidden_size
        self.num_heads = num_heads
        self.head_dim = hidden_size // num_heads
        self.is_cross_attention = is_cross_attention

        # 注意：T5去掉了所有偏置项（bias=False）
        self.q = nn.Linear(hidden_size, hidden_size, bias=False)
        self.k = nn.Linear(hidden_size, hidden_size, bias=False)
        self.v = nn.Linear(hidden_size, hidden_size, bias=False)
        self.o = nn.Linear(hidden_size, hidden_size, bias=False)

        # 只有自注意力层的第一层有相对位置偏置（跨层共享）
        self.relative_position_bias = (
            T5RelativePositionBias(num_heads) if has_relative_bias else None
        )

    def forward(self, hidden_states, key_value_states=None,
                attention_mask=None, position_bias=None):
        batch_size, seq_len, _ = hidden_states.shape

        # 投影
        q = self.q(hidden_states).view(batch_size, -1, self.num_heads, self.head_dim).transpose(1, 2)

        if self.is_cross_attention and key_value_states is not None:
            k = self.k(key_value_states).view(batch_size, -1, self.num_heads, self.head_dim).transpose(1, 2)
            v = self.v(key_value_states).view(batch_size, -1, self.num_heads, self.head_dim).transpose(1, 2)
        else:
            k = self.k(hidden_states).view(batch_size, -1, self.num_heads, self.head_dim).transpose(1, 2)
            v = self.v(hidden_states).view(batch_size, -1, self.num_heads, self.head_dim).transpose(1, 2)

        # 注意力分数（T5不除以sqrt(d_k)，用相对位置偏置代替缩放）
        scores = torch.matmul(q, k.transpose(-2, -1))

        # 加相对位置偏置
        if position_bias is None and self.relative_position_bias is not None:
            position_bias = self.relative_position_bias(seq_len, k.shape[2])
        if position_bias is not None:
            scores = scores + position_bias

        # 注意力掩码
        if attention_mask is not None:
            scores = scores + attention_mask

        weights = torch.softmax(scores.float(), dim=-1).to(scores.dtype)

        # 输出
        output = torch.matmul(weights, v)
        output = output.transpose(1, 2).contiguous().view(batch_size, -1, self.hidden_size)
        return self.o(output), position_bias


class T5Block(nn.Module):
    """T5 Transformer块"""

    def __init__(self, hidden_size: int, num_heads: int,
                 intermediate_size: int, is_decoder: bool = False,
                 has_relative_bias: bool = True):
        super().__init__()
        self.is_decoder = is_decoder

        # 自注意力
        self.self_attn = T5Attention(hidden_size, num_heads,
                                      has_relative_bias=has_relative_bias)
        self.self_attn_norm = T5RMSNorm(hidden_size)

        # 交叉注意力（仅解码器）
        if is_decoder:
            self.cross_attn = T5Attention(hidden_size, num_heads,
                                           is_cross_attention=True,
                                           has_relative_bias=False)
            self.cross_attn_norm = T5RMSNorm(hidden_size)

        # 前馈网络（使用GeGLU激活）
        self.ff_norm = T5RMSNorm(hidden_size)
        self.wi_0 = nn.Linear(hidden_size, intermediate_size, bias=False)
        self.wi_1 = nn.Linear(hidden_size, intermediate_size, bias=False)
        self.wo = nn.Linear(intermediate_size, hidden_size, bias=False)

    def forward(self, hidden_states, encoder_hidden_states=None,
                attention_mask=None, encoder_attention_mask=None,
                position_bias=None, encoder_position_bias=None):

        # 1. 自注意力（Pre-Norm）
        normed = self.self_attn_norm(hidden_states)
        attn_output, position_bias = self.self_attn(
            normed, attention_mask=attention_mask, position_bias=position_bias
        )
        hidden_states = hidden_states + attn_output

        # 2. 交叉注意力（仅解码器）
        if self.is_decoder and encoder_hidden_states is not None:
            normed = self.cross_attn_norm(hidden_states)
            cross_output, encoder_position_bias = self.cross_attn(
                normed, key_value_states=encoder_hidden_states,
                attention_mask=encoder_attention_mask,
                position_bias=encoder_position_bias
            )
            hidden_states = hidden_states + cross_output

        # 3. 前馈网络（GeGLU: gated linear unit with GELU）
        normed = self.ff_norm(hidden_states)
        # GeGLU: output = GELU(W1*x) ⊙ (W2*x)
        gate = torch.nn.functional.gelu(self.wi_0(normed))
        proj = self.wi_1(normed)
        ff_output = self.wo(gate * proj)
        hidden_states = hidden_states + ff_output

        return hidden_states, position_bias, encoder_position_bias


print("T5核心组件定义完成")
print(f"T5RMSNorm: 去掉均值中心化的简化层归一化")
print(f"T5RelativePositionBias: 相对位置偏置，支持分桶映射")
print(f"T5Attention: 无偏置项的注意力层")
print(f"T5Block: Pre-Norm结构，包含自注意力+交叉注意力+GeGLU前馈")
```

def t5_generate(task_prefix: str, input_text: str,
                max_new_tokens: int = 128,
                num_beams: int = 4) -> str:
    """
    使用T5执行各种任务

    Args:
        task_prefix: 任务前缀，如 "translate English to German: "
        input_text: 输入文本
        max_new_tokens: 最大生成长度
        num_beams: 束搜索宽度

    Returns:
        生成的文本
    """
    # 构造输入
    full_input = task_prefix + input_text
    inputs = tokenizer(full_input, return_tensors="pt",
                       max_length=512, truncation=True)

    with torch.no_grad():
        outputs = model.generate(
            input_ids=inputs["input_ids"],
            attention_mask=inputs["attention_mask"],
            max_new_tokens=max_new_tokens,
            num_beams=num_beams,
            early_stopping=True
        )

    # 解码输出
    return tokenizer.decode(outputs[0], skip_special_tokens=True)

class MultiTaskT5Dataset(Dataset):

        self.max_input_length = max_input_length
        self.max_target_length = max_target_length

        # 多任务训练数据（实际使用时应从文件加载）
        self.data = [

            {
                "input": "translate English to French: Hello, how are you?",

            },
            {
                "input": "translate English to French: The weather is nice today.",
                "target": "Le temps est agréable aujourd'hui."
            },
            # 情感分类任务
            {
                "input": "sst2 sentence: This movie is absolutely fantastic!",
                "target": "positive"
            },
            {
                "input": "sst2 sentence: The film was a complete waste of time.",
                "target": "negative"
            },
            # 摘要任务
            {
                "input": "summarize: Scientists have discovered a new species of deep-sea fish in the Pacific Ocean. The fish, found at depths of over 3000 meters, has unique bioluminescent properties.",
                "target": "New deep-sea fish species with bioluminescence discovered in Pacific."
            },
            # 问答任务
            {
                "input": "question: What is the capital of France? context: France is a country in Western Europe. Its capital city is Paris, which is also the largest city.",
                "target": "Paris"
            },
        ]

    def __len__(self):
        return len(self.data)

    def __getitem__(self, idx):
        item = self.data[idx]

        # 编码输入
        input_encoding = self.tokenizer(
            item["input"],
            max_length=self.max_input_length,
            padding="max_length",
            truncation=True,
            return_tensors="pt"
        )

        # 编码目标（标签中的padding用-100表示，计算loss时忽略）
        target_encoding = self.tokenizer(
            item["target"],
            max_length=self.max_target_length,
            padding="max_length",
            truncation=True,
            return_tensors="pt"
        )
        labels = target_encoding["input_ids"].squeeze()
        labels[labels == self.tokenizer.pad_token_id] = -100

        return {
            "input_ids": input_encoding["input_ids"].squeeze(),
            "attention_mask": input_encoding["attention_mask"].squeeze(),
            "labels": labels
        }


def fine_tune_t5(model_name: str = "t5-small",
                 num_epochs: int = 3,
                 learning_rate: float = 5e-4,
                 batch_size: int = 4):
    """
    T5多任务微调
    """
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    print(f"使用设备: {device}")

    # 加载模型和分词器
    tokenizer = T5Tokenizer.from_pretrained(model_name)
    model = T5ForConditionalGeneration.from_pretrained(model_name)
    model = model.to(device)

    # 准备数据
    dataset = MultiTaskT5Dataset(tokenizer)
    dataloader = DataLoader(dataset, batch_size=batch_size, shuffle=True)

    # 优化器（T5原论文使用Adafactor，这里用AdamW作示例）
    optimizer = AdamW(model.parameters(), lr=learning_rate)

    # 训练循环
    model.train()
    for epoch in range(num_epochs):
        total_loss = 0.0
        for batch_idx, batch in enumerate(dataloader):
            input_ids = batch["input_ids"].to(device)
            attention_mask = batch["attention_mask"].to(device)
            labels = batch["labels"].to(device)

            # 前向传播（T5的generate和训练模式共用同一模型）
            outputs = model(
                input_ids=input_ids,
                attention_mask=attention_mask,
                labels=labels
            )

            loss = outputs.loss
            total_loss += loss.item()

            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            # 梯度裁剪，防止梯度爆炸
            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
            optimizer.step()

            if batch_idx % 10 == 0:
                print(f"Epoch {epoch+1}, Batch {batch_idx}, Loss: {loss.item():.4f}")

        avg_loss = total_loss / len(dataloader)
        print(f"Epoch {epoch+1} 完成, 平均损失: {avg_loss:.4f}")

    return model, tokenizer

- (b) 计算在 $T=2$ 时，任务B被采样到的概率

## 练习答案

**题目1答案：**

Text-to-Text框架的核心思想是：**将所有NLP任务统一转化为"输入文本 → 输出文本"的生成问题**，使用同一套模型架构、损失函数和解码策略处理所有任务。

文本分类（以SST-2情感分类为例）的转化方式：
- 原始格式：输入句子 → 输出类别概率向量 → argmax得到类别
- Text-to-Text格式：`"sst2 sentence: The movie is great!"` → `"positive"`

任务前缀的作用：
1. **任务标识**：告知模型当前需要执行哪种操作（通过前缀区分翻译/分类/摘要等）
2. **无需结构改变**：不需要为每个任务添加特殊的输出头，仅靠文本传达任务信息
3. **灵活切换**：推理时通过修改前缀即可切换任务
4. **多任务学习桥梁**：混合训练时，模型通过前缀学会条件性执行不同任务

**题目2答案：**

| 对比维度 | BERT的MLM | T5的Span Corruption |
|---------|-----------|-------------------|
| **掩盖粒度** | 单个token（15%的token被随机掩盖） | 连续片段（span），默认平均长度3个token |
| **哨兵Token** | 用`[MASK]`替换每个被掩盖token，多个掩盖位置用多个`[MASK]` | 整个span只用一个哨兵Token（`<extra_id_0>`等）替换，哨兵Token在输出端也作分隔符 |
| **目标序列长度** | 与输入等长（对所有位置预测，掩盖位置的损失有效） | 远短于输入（只生成被掩盖的片段内容） |

Span Corruption的优势：目标序列短，每步生成的信息密度更高，训练效率提升；同时需要学习理解跨片段的上下文关系，对生成能力要求更高。

**题目3答案：**

**(a) 分桶设计的原因：**

对注意力机制来说，**近距离位置关系更细粒度、更重要**，而**远距离位置关系可以粗略处理**。

- **小距离线性分桶**：位置1和位置2的差别（相邻token）对句法分析非常关键；位置3和位置4的差别（隔两个token）也很重要。线性分桶确保每个近距离都有独立的偏置值，精确建模。
- **大距离对数分桶**：位置100和位置101的差别与位置200和位置201的差别，对模型来说几乎没有区别——都表示"相距很远"。对数分桶将大范围的远距离压缩到少数几个桶，既节省参数，又允许模型泛化到训练时未见过的更长序列。

数学上，设最大精确距离为 $r$，则 $[r, D]$ 范围的距离被压缩到 $r$ 个桶中，映射函数为 $\lfloor r + \log(d/r) / \log(D/r) \cdot r \rfloor$，其中 $D$ 为最大距离。

**(b) 跨层共享位置偏置的好处：**

1. **减少参数量**：若每层都有独立偏置，则参数量为 $L \times H \times B$（$L$层，$H$头，$B$桶）；共享后降为 $H \times B$
2. **正则化效果**：共享参数迫使模型学习到任务无关的、更通用的位置关系，防止过拟合
3. **实践有效**：T5论文实验表明，共享位置偏置与每层独立偏置效果相当，但参数更少

**题目4答案：**

设任务A数据量 $n_A = 1,000,000$，任务B数据量 $n_B = 10,000$，总量 $N = 1,010,000$。

温度采样公式：$p_i \propto (n_i / N)^{1/T}$

**(a) T=1时任务B的采样概率：**

$$p_B(T=1) = \frac{n_B / N}{n_A/N + n_B/N} = \frac{n_B}{N} = \frac{10000}{1010000} \approx 0.0099 \approx 1\%$$

**(b) T=2时任务B的采样概率：**

先计算各任务的权重：

$$w_A = (1000000/1010000)^{0.5} \approx (0.9901)^{0.5} \approx 0.9950$$

$$w_B = (10000/1010000)^{0.5} \approx (0.0099)^{0.5} \approx 0.0995$$

归一化：$p_B(T=2) = 0.0995 / (0.9950 + 0.0995) \approx 0.0909 \approx 9.1\%$

**(c) T=2的优势分析：**

- 在 $T=1$ 时，任务B只占约1%的训练样本，模型在整个训练过程中见到该任务的次数极少，会导致任务B严重欠拟合
- 在 $T=2$ 时，任务B占约9%，得到更充分的训练
- $T \to \infty$ 时趋向50%，这又会让任务A的数据被过度稀释，模型反复看相同的任务B样本导致过拟合

$T=2$ 在两种极端之间取得平衡：小任务得到足够曝光，大任务的知识优势得以保留，两者共同提升多任务性能。

**题目5答案：**

**实验设计：**

研究假设：在文本摘要任务上，经过Span Corruption预训练的T5优于经过MLM预训练的T5（编码器-解码器架构）。

控制变量设置：
| 变量 | Span Corruption组 | MLM组 |
|------|-----------------|-------|
| 架构 | T5编码器-解码器 | T5编码器-解码器（相同） |
| 预训练数据 | C4（相同） |C4（相同）|
| 预训练步数 | 500K步 | 500K步（相同）|
| 预训练目标 | Span Corruption（15%，span=3） | MLM（15%单token）|
| 微调数据 | CNN/DailyMail摘要数据集 | 相同 |
| 微调步数 | 10K步 | 相同 |
| 超参数 | 相同 | 相同 |

**评估指标：**
- **ROUGE-1/2/L**：标准摘要评估指标，衡量与参考摘要的n-gram重叠度
- **BERTScore**：基于语义相似度，比ROUGE更鲁棒（对同义改写不敏感）
- 选择多个指标的原因：单一指标可能有偏，ROUGE倾向于奖励表面相似，BERTScore更关注语义

**潜在混淆变量：**
1. **预训练目标对Decoder的影响**：MLM只训练编码器，Decoder在预训练时没有监督信号（或用掩码LM），这可能不公平。需要确保两组的Decoder预训练方式相同。
2. **有效训练token数量**：Span Corruption的目标序列比MLM短（因为Sentinel Token替换了span），等步数下实际处理的token数不同。可以按token数而非步数对齐。
3. **任务对齐性**：摘要本质上是seq2seq生成，Span Corruption本身也是生成任务，天然更对齐。需额外测试在纯NLU任务（如分类）上的对比，验证MLM是否在理解型任务上反超。

**统计显著性分析：**
- 对CNN/DailyMail测试集（约11,000条）进行**bootstrap自助法**，重采样1000次，计算ROUGE分数的95%置信区间
- 若两组置信区间无重叠，结论有统计显著性
- 此外进行**配对t检验**（paired t-test），验证逐样本的分数差异
- 建议在3个随机种子下重复实验，报告均值和标准差，排除随机性影响
