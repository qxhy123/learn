# 附录 A2：核心结构速查表

| 概念 | 作用 | 常见误区 |
|------|------|----------|
| 路径 | 描述如何找到目标对象 | 把路径误当成对象本体 |
| 目录项 | 名字到对象编号的映射 | 把目录项和 inode 混为一谈 |
| dentry | VFS 路径查找的重要对象/缓存 | 误以为它等于磁盘目录项 |
| negative dentry | 缓存“这个名字当前不存在” | 误以为缓存只缓存成功查找 |
| inode | 描述对象元数据与数据位置 | 误以为它保存文件名 |
| open file description | 表示打开中的文件状态 | 误以为等于 inode 或 fd 本身 |
| 文件描述符 | 进程引用打开文件的索引 | 误以为 fd 就是文件本身 |
| superblock | 文件系统实例全局信息 | 误以为每个文件都有一个 superblock |
| data block / extent | 保存文件数据或数据范围映射 | 误以为内容总是连续存放 |
| page cache | 缓存文件页，提高 I/O 性能 | 误以为写入 cache 等于已落盘 |
| writeback | 后台把脏页回写到底层存储 | 误以为写入返回就等于回写完成 |
| journal | 帮助关键更新在崩溃后恢复一致 | 误以为它保证所有最新数据不丢 |
| VFS | 统一不同文件系统接口 | 误以为所有文件系统底层实现相同 |
| mount | 把文件系统实例接入目录树 | 误以为只是“显示一个目录” |
| mount namespace | 决定一个进程看到哪棵挂载树 | 误以为路径脱离上下文也唯一 |
| overlayfs | 组合多层目录视图 | 误以为容器一定复制整套根文件系统 |
| whiteout | 在联合视图中遮蔽下层对象 | 误以为删除下层对象不需要特殊表示 |
| close-to-open | NFS 常见的一致性直觉 | 误以为所有客户端始终强一致 |
| FUSE | 用户态实现文件系统接口的方式 | 误以为文件系统必须完全驻留内核 |
