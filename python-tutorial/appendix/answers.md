# 附录C：练习答案汇总

本附录为全书27章的练习题提供索引、要点提示和解题思路。每章设有5道练习题，共计135道，分布于三个难度级别。

> **说明**：本附录不提供完整代码答案，而是给出每道题的核心思路和关键知识点。完整答案请参阅各章末尾的「练习答案」部分。

## 如何有效使用练习题

### 做题策略

1. **独立思考优先**：拿到题目后，先独立思考至少15分钟，不要立即翻答案。
2. **动手编码**：在编辑器中实际运行代码，观察输出，不要只在脑海中"模拟"。
3. **从错误中学习**：运行失败时，认真阅读错误信息，错误信息本身就是最好的提示。
4. **对比答案**：完成后将自己的解法与参考答案对比，关注代码风格和效率差异。
5. **举一反三**：尝试修改题目条件，探索代码在不同输入下的行为。

### 复习建议

- **基础题**做错：回到章节正文重新阅读对应小节
- **中级题**做错：尝试把问题拆分为更小的步骤，逐步实现
- **提高题**做错：先完成基础和中级题，再回来挑战；参考答案后理解思路，隔天独立重写

### 进度参考

| 学习路径 | 建议完成题目 |
|---------|------------|
| 零基础入门 | 所有基础题 + 中级题，提高题选做 |
| 快速进阶 | 所有题目，基础题可快速浏览 |
| 深度学习导向 | 第5、8-27章全部题目 |

## 常见错误提示

### 语法类错误

| 错误类型 | 典型症状 | 解决方向 |
|---------|---------|---------|
| `IndentationError` | 缩进不一致 | 统一使用4个空格，不混用Tab |
| `SyntaxError` | 括号未闭合、冒号缺失 | 检查循环、函数、类定义末尾的冒号 |
| `NameError` | 变量未定义就使用 | 检查变量名拼写，确认赋值在使用之前 |
| `TypeError` | 类型不匹配的操作 | 用`type()`检查变量类型，必要时做类型转换 |

### 逻辑类错误

| 错误类型 | 典型症状 | 解决方向 |
|---------|---------|---------|
| 差一错误 | 循环多一次或少一次 | 检查`range()`的起止参数，用小数据手动验证 |
| 可变默认参数 | 函数多次调用结果异常 | 默认参数使用`None`，函数体内初始化 |
| 浅拷贝陷阱 | 修改副本影响原列表 | 使用`copy.deepcopy()`或切片`[:]`创建深拷贝 |
| 浮点精度 | 数值比较返回意外结果 | 使用`math.isclose()`代替`==`比较浮点数 |

### 深度学习类错误

| 错误类型 | 典型症状 | 解决方向 |
|---------|---------|---------|
| 维度不匹配 | `RuntimeError: size mismatch` | 用`.shape`打印每步张量形状 |
| 梯度未清零 | 训练损失异常波动 | 每个batch前调用`optimizer.zero_grad()` |
| 设备不一致 | `Expected all tensors on same device` | 统一将模型和数据移到同一设备 |
| 数值溢出 | 损失变为`nan`或`inf` | 检查学习率，使用梯度裁剪 |

| 练习5 | 【提高】 | 完整部署流水线 | 训练→评估→导出→量化→服务→监控的端到端流程；A/B测试框架；性能基准测试`torch.utils.benchmark` |

**解题提示**：`torch.jit.trace()`不能处理依赖输入的控制流（如`if x.shape[0] > 1:`），这种情况必须使用`torch.jit.script()`。

### 第25章：asyncio基础与结构化并发

**章节链接**：[asyncio基础与结构化并发](../part9-asyncio/25-asyncio-foundations.md)

**本章主题**：协程、事件循环内部模型、selector/reactor、Task、Future、loop API、`asyncio.Event`、TaskGroup、取消传播

|------|------|---------|-----------|
| 练习1 | 【基础】 | I/O 密集 vs CPU 密集 | `asyncio` 适合等待型任务；CPU 密集任务会卡住 loop |
| 练习2 | 【基础】 | event loop 一次 tick 的结构 | ready queue、timer、I/O watcher、任务恢复 |
| 练习3 | 【中级】 | `selector / reactor` + 平台后端 | `epoll / kqueue / IOCP` 处于事件通知层；loop 在其上做统一调度 |

| 练习5 | 【提高】 | 带 loop / Task / Event 的异步聚合器 | 启动屏障、并发抓取、Future / Task 生命周期协调 |

**章节链接**：[asyncio网络编程与异步服务实战](../part9-asyncio/26-asyncio-networking-and-services.md)

**本章主题**：`start_server()`、stream API、transport/protocol、背压、服务生命周期、`Event`、signal、优雅关闭、异步服务结构

|------|------|---------|-----------|

**章节链接**：[asyncio极端复杂场景实战](../part9-asyncio/27-asyncio-extreme-scenarios.md)

**本章主题**：deadline 预算、微批处理、多租户限流、取消风暴、复杂编排、异步推理网关

|------|------|---------|-----------|
