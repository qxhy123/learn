# 领域驱动设计（DDD）从零到高阶教程

> 一套系统、深入的DDD学习路径，覆盖战略设计、战术设计、架构模式与高阶实践

---

## 教程定位

本教程面向有一定编程基础，希望系统掌握领域驱动设计（Domain-Driven Design，DDD）的开发者。内容从"DDD是什么"出发，逐步深入到战略设计、战术构建块、架构模式，最终覆盖微服务、CQRS、事件溯源等高阶主题。

**代码示例语言**：Python（部分章节含伪代码对照）

**学习时长**：系统学习约40~60小时

## 各部分概述

### 第一部分：基础认知
理解DDD的核心哲学——软件复杂度来自于业务复杂度，解决之道是让代码模型与业务模型深度对齐。

| 章节 | 主题 | 关键概念 |
|------|------|---------|
| 01 | 什么是DDD？ | 软件复杂度、模型驱动设计 |
| 02 | 核心概念 | 领域、子域、限界上下文 |
| 03 | 统一语言 | Ubiquitous Language、语言边界 |
| 04 | 战略与战术 | 两层设计的关系与职责 |

### 第二部分：战略设计
在大型系统中划定边界，明确不同子系统的职责与协作关系。

| 章节 | 主题 | 关键概念 |
|------|------|---------|
| 05 | 限界上下文 | BC定义、BC内部一致性 |
| 06 | 上下文映射 | 集成模式、防腐层、共享内核 |
| 07 | 子域类型 | 核心域、支撑域、通用域 |
| 08 | 事件风暴 | 协作建模、事件发现 |

### 第三部分：战术设计
在单个限界上下文内部，用构建块精确表达业务规则。

| 章节 | 主题 | 关键概念 |
|------|------|---------|
| 09 | 实体与值对象 | 标识、不变性、相等性 |
| 10 | 聚合与聚合根 | 一致性边界、不变量保护 |
| 11 | 领域事件 | 事件的语义、发布与订阅 |
| 12 | 仓储 | 持久化抽象、集合语义 |
| 13 | 领域服务 | 无状态领域逻辑 |
| 14 | 应用服务 | 用例编排、事务边界 |
| 15 | 工厂 | 复杂对象创建 |

### 第四部分：架构模式
将DDD思想落地到具体架构中。

| 章节 | 主题 | 关键概念 |
|------|------|---------|
| 16 | 分层架构 | 依赖方向、各层职责 |
| 17 | 六边形架构 | 端口与适配器、可测试性 |
| 18 | CQRS | 命令查询分离、读写模型 |
| 19 | 事件溯源 | 事件即状态、时间旅行 |

### 第五部分：高阶实践
在真实复杂系统中运用DDD。

| 章节 | 主题 | 关键概念 |
|------|------|---------|
| 20 | DDD与微服务 | 服务边界、自治性 |
| 21 | Saga模式 | 分布式事务、补偿 |
| 22 | 防腐层 | 翻译层、遗留系统集成 |
| 23 | 测试策略 | 领域测试金字塔 |
| 24 | 综合案例 | 电商系统完整设计 |

## 如何使用本教程

1. **按顺序阅读**：各章节有依赖关系，建议按序学习
2. **动手写代码**：每章的代码示例都应自己运行一遍
3. **结合项目**：尝试将所学应用到自己的实际项目中
4. **反复迭代**：DDD需要在实践中逐渐体会，多次阅读效果更好

> "好的设计不是从一开始就完美，而是在探索中不断精炼。" —— Eric Evans
