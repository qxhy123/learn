# 附录A：AI Infra 术语表

| 术语 | 简要解释 |
|------|----------|
| AI Infra | 承载 AI 数据、训练、评测、部署、推理、监控与治理的基础设施体系 |
| GPU | 适合高吞吐并行计算的加速设备，常用于训练和推理 |
| 显存 | GPU 上的高速内存，决定模型、batch、缓存能否放下 |
| Checkpoint | 训练过程中的可恢复状态，通常包含模型参数、优化器状态和 step 信息 |
| 模型包 | 面向部署的模型产物，通常包含权重、配置、tokenizer 与推理元数据 |
| 数据并行 | 多个设备各自处理不同 batch 分片，并同步梯度 |
| 张量并行 | 将同一层内部计算切分到多个设备上执行 |
| 流水线并行 | 将模型不同层分配到不同设备或阶段执行 |
| FSDP / ZeRO | 对参数、梯度、优化器状态进行切分以降低单设备显存压力的训练技术族 |
| NCCL | 常用于 GPU 间通信的集合通信库 |
| All-reduce | 多个进程聚合数据并把结果分发给所有进程的通信操作 |
| KV Cache | LLM 推理中缓存历史 key/value，减少 decode 阶段重复计算 |
| Continuous batching | LLM 服务中动态组织正在生成的请求，提高 decode 吞吐的批处理方式 |
| PagedAttention | 将 KV Cache 分块管理以降低显存碎片和预分配浪费的思路 |
| SLO | 服务等级目标，用于定义可用性、延迟、错误率等目标 |
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| Embedding | 将文本、图片等对象映射为向量表示 |
| 向量索引 | 支持近似最近邻检索的数据结构或服务 |
| 灰度发布 | 让新模型或新服务先接收小比例流量，再逐步放量 |
| 成本归因 | 将 GPU、存储、网络等资源成本归属到团队、项目、任务或模型 |
