# 从零到高阶的 PowerShell 教程

## 项目简介

很多人第一次接触 PowerShell 时，会把它误判成“Windows 版 Bash”或者“把命令换了一套名字的 CMD”。

这会直接导致三个典型问题：

- 会抄命令，但不知道为什么有些输出能继续传给下一个命令，有些不行
- 会写一段能跑的脚本，但一遇到错误、权限、远程执行就开始不稳
- 会完成一次性任务，但很难把它整理成可复用、可维护、可安全执行的工具

这套教程的目标不是堆 cmdlet 清单，而是帮你建立一条稳定主线：

1. PowerShell 的核心对象不是“文本行”，而是**对象**
2. 真正决定效率的，不是背了多少命令，而是你是否理解**对象、管道、脚本、自动化边界**
3. 高阶能力不只是“功能更多”，而是能在批量操作和系统管理里保持**可观察、可恢复、可控**

如果你能持续从这三个角度看命令、脚本和自动化任务，PowerShell 就会从“偶尔能用”变成“能交付工具”的语言。

---

## 目标受众

- 完全没有 Shell 经验、希望从零开始的读者
- 会一点 CMD 或 Bash，但不理解 PowerShell 对象模型的开发者
- 需要把重复系统操作变成脚本的运维、测试和平台工程师
- 希望从“会跑命令”升级到“会写可复用工具”的进阶用户

## 学习路径建议

### 3 天入门闭环

1. 完成第 1-3 章
2. 每学一个命令都同时跑一次结果查看和一次 `Get-Member`
3. 验收标准：你能解释“为什么 PowerShell 管道里传的是对象，而不是文本”

### 5 天脚本化进阶

1. 完成第 4-5 章
2. 用函数重写一次你平时重复执行的命令序列
3. 验收标准：你能写出带参数、有默认值、能复用的脚本

### 4 天自动化实战

1. 完成第 6-7 章
2. 选一个管理任务，先做只读版，再加 `-WhatIf`，最后再做实际执行版
3. 验收标准：你能先观察对象和风险，再决定是否批量执行

### 3 天高阶收束

1. 完成第 8 章
2. 回看附录 A、B、D
3. 验收标准：你能解释高级函数、模块边界、兼容性和安全约束之间的关系

## 前置要求

- **必需**：基础命令行操作能力
- **推荐**：一台可运行 PowerShell 的 Windows、macOS 或 Linux 机器
- **推荐**：有一点文件、进程、服务等系统概念会更轻松

如果你是完全新手，也可以直接开始。本教程会先建立观察习惯，再逐步进入脚本化和自动化。

---

## 环境与练习约定

推荐环境：

```powershell
$PSVersionTable
Get-Command pwsh, powershell -ErrorAction SilentlyContinue
```

建议你在练习时固定保留三种观察动作：

```powershell
Get-Help <CommandName> -Examples
<CommandName> | Get-Member
<CommandName> | Select-Object -First 5
```

这三步会强迫你养成正确习惯：

- 先看官方帮助
- 再看对象结构
- 最后再决定如何筛选、排序、格式化或写入脚本

---

## 如何使用本教程

1. **先理解对象，再背命令。**
2. **不要把 `Format-*` 当成数据处理命令。**
3. **在批量操作前，先做只读观察或 `-WhatIf` 演练。**
4. **每学完一章，都亲手做一次最小练习。**
5. **把“先看帮助、再看对象”变成肌肉记忆。**

---

## 教程特色

- **结构扁平**：只有 `chapters/`、`labs/`、`appendix/` 三个主目录
- **从对象心智模型切入**：避免把 PowerShell 学成命令字典
- **强调自动化安全**：把 `-WhatIf`、错误处理、日志和幂等当成主线内容
- **兼顾 Windows 管理与跨平台视角**：既不丢掉系统管理场景，也不忽视 PowerShell 7
- **练习可重复**：每章都有可直接动手的最小任务和对应答案索引
