# 前言：如何学习 PowerShell，而不是只背命令

## 这门教程到底想解决什么问题

很多人学 PowerShell 时会卡在两个极端：

- 一种是把它当成“Windows 命令大全”，学完一堆 `Get-*`、`Set-*`，但不会组织脚本
- 另一种是上来就学语法、函数、模块，却没先理解对象和管道，结果越学越乱

这套教程的核心目标不是让你记住很多命令，而是让你形成一套稳定判断：

1. 这个命令返回的到底是什么对象
2. 这个对象接下来应该继续走管道，还是应该先选属性、再格式化
3. 这个脚本现在只是能跑，还是已经达到了可复用、可调试、可安全执行

---

## 为什么 PowerShell 值得单独学

如果你已经接触过 Bash、CMD 或 Python，PowerShell 仍然值得单独学，因为它把几件事组合到了一起：

- 既是交互式 Shell，也是脚本语言
- 既适合系统管理，也适合批量数据处理
- 既能处理文本，也能直接处理结构化对象
- 既能做临时操作，也能演化成模块和工具

真正重要的差异不在“命令长得不一样”，而在于：

> PowerShell 默认传递的是对象，不是文本行。

这句话如果真的理解了，后面很多设计都会顺畅很多。

---

## 学习顺序为什么这么安排

本教程故意不按“变量 -> if -> for -> 函数 -> 模块”这种传统语法顺序起步。

原因很简单：

- PowerShell 初学者最容易卡住的不是语法，而是**对象和管道**
- 你如果不理解 `Get-Member`、属性、方法、`Select-Object`、`Where-Object`，后面写脚本时会一直在猜
- 只有先看懂对象，后面写条件、循环、函数时，才知道自己到底在处理什么

所以本教程会先让你学会观察，再让你学会组织。

---

## 建议你坚持的三个固定动作

以后你见到任何一个新命令，先做三件事：

```powershell
Get-Help <CommandName> -Examples
<CommandName> | Get-Member
<CommandName> | Select-Object -First 5
```

这三步分别解决三类问题：

- 帮助系统告诉你：这个命令是干什么的
- `Get-Member` 告诉你：它输出的对象长什么样
- `Select-Object` 告诉你：你实际拿到了哪些值，哪些属性值得继续处理

很多“脚本写着写着不对劲”的根源，都是因为少了这三步。

## 本教程的训练方式

你会频繁看到四类内容：

- **最小示例**：让你先跑起来
- **对象观察**：帮助你知道管道里到底在流什么
- **常见误区**：避免把 PowerShell 学成危险脚本语言
- **本章练习**：把“会看”变成“会做”

建议你按下面的节奏学习：

1. 先通读一节，理解它解决的问题
2. 手动把示例敲一遍，不要复制粘贴
3. 对输出再跑一次 `Get-Member`
4. 完成最小练习后，再看答案索引核对

## 关于环境和版本

建议优先使用 PowerShell 7+ 学习，因为它跨平台、更新快、长期更适合作为现代默认环境。

但你也需要知道：

- 很多 Windows 管理场景仍然会碰到 Windows PowerShell 5.1
- 某些 Provider、模块和管理能力带有明显平台差异
- 执行策略、注册表 Provider、某些 remoting 场景主要发生在 Windows

因此，本教程会把“核心语法与对象模型”和“平台相关特性”区分开讲。

---

## 你学完之后应具备什么能力

如果你完整学完这套教程，你至少应能做到：

- 独立查帮助并探索未知命令
- 用对象管道处理系统信息，而不是只靠文本拼接
- 写带参数、可复用、可调试的 PowerShell 脚本
- 在批量操作前先判断风险、先观察状态、先做只读验证
- 把脚本整理成函数、模块和团队可复用工具

如果你达到这一步，PowerShell 对你来说就不再是“遇到时临时查一条命令”的工具，而是一套真正可依赖的自动化语言。
