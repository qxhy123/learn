# 第1章：双指针是什么

## 1.1 从一道题目开始

给定一个数组 `nums`，找出其中和为目标值 `target` 的两个数的下标。

最直接的暴力解法：

```python
def two_sum_brute(nums, target):
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []
```

时间复杂度 O(n²)。当 n = 100,000 时，需要约 **50亿次**操作——这在实际中完全不可接受。

**问题在哪里？** 我们在用两层循环枚举"所有可能的配对"，其中大量配对是无意义的。

双指针的核心思想：**利用某种有序性或单调性，让两个指针协同移动，在一次扫描中完成原本需要两次的工作。**

---

## 1.2 指针的本质：搜索空间的游标

"指针"在这里不是 C 语言的内存指针，而是**数组下标**或**链表节点引用**。

把搜索空间想象成一个二维矩阵：对于长度为 n 的数组，所有 (i, j) 对构成一个 n×n 的方格。

```
j →   0  1  2  3  4
i ↓
0     ×  ✓  ✓  ✓  ✓
1        ×  ✓  ✓  ✓
2           ×  ✓  ✓
3              ×  ✓
4                 ×
```

暴力法遍历右上三角的所有格子：O(n²)。

双指针的目标：**沿某条路径只访问 O(n) 个格子，同时保证答案不被遗漏。**

这条路径能否存在，取决于问题是否具备**单调性**。

---

## 1.3 双指针的三种形态

### 形态一：对撞指针（Opposite Direction）

```
[  1,  2,  3,  4,  5,  6  ]
  ↑                    ↑
  left                right
```

- `left` 从左向右移动，`right` 从右向左移动
- 两指针相向而行，最终"碰撞"
- **适用场景**：有序数组、回文判断、两端逼近

### 形态二：同向双指针（Same Direction）

```
[  1,  2,  3,  4,  5,  6  ]
  ↑    ↑
 slow fast
```

- `slow` 和 `fast` 都从左向右移动，但速度不同
- **适用场景**：数组去重、滑动窗口、移除元素

### 形态三：快慢指针（Fast & Slow）

```
1 → 2 → 3 → 4 → 5 → 3（成环）
↑        ↑
slow    fast
```

- 主要用于链表
- `fast` 每次走两步，`slow` 每次走一步
- **适用场景**：环形链表检测、找中点、倒数第 K 节点

---

## 1.4 为什么双指针是 O(n) 的？

关键在于：**每个元素最多被访问常数次**。

以对撞指针为例：
- `left` 只向右移动，最多移动 n 次
- `right` 只向左移动，最多移动 n 次
- 总移动次数 ≤ 2n，即 O(n)

以滑动窗口为例：
- `right` 扫过数组一次：n 次移动
- `left` 追赶 `right`，也最多移动 n 次
- 总移动次数 ≤ 2n，即 O(n)

**数学本质**：两个指针的移动次数之和被数组长度限制住了，因为指针只能单向移动且不能越界。

---

## 1.5 双指针成立的前提：单调性

双指针不是万能的，它依赖于问题具备某种**单调性**：

> 当一个指针移动时，另一个指针的最优位置也沿同一方向变化。

**例子（两数之和，有序数组）**：
- 设 `left=0, right=n-1`，当前和 `s = nums[left] + nums[right]`
- 如果 `s > target`：`right--`（减小右边的数）
- 如果 `s < target`：`left++`（增大左边的数）

**单调性保证**：当 `left` 增大时，满足条件的 `right` 不会增大（只会减小或不变）。这使得我们可以"两指针协同推进"而不遗漏答案。

如果没有单调性，双指针就可能漏掉答案，这时需要其他方法（如哈希表）。

---

## 1.6 双指针 vs 哈希表

两者都能将某类 O(n²) 问题优化到 O(n)，但适用场景不同：

| 方法 | 时间 | 空间 | 前提 | 典型场景 |
|------|------|------|------|----------|
| 双指针 | O(n) | O(1) | 数组有序 或 具备单调性 | 有序数组查找、滑动窗口 |
| 哈希表 | O(n) | O(n) | 无需有序 | 无序数组查找、计数 |

```python
# 无序数组的两数之和：用哈希表
def two_sum_hash(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []

# 有序数组的两数之和：用双指针
def two_sum_sorted(nums, target):
    left, right = 0, len(nums) - 1
    while left < right:
        s = nums[left] + nums[right]
        if s == target:
            return [left, right]
        elif s < target:
            left += 1
        else:
            right -= 1
    return []
```

**选择原则**：
- 数组已排序，或排序代价可接受 → 优先双指针（O(1) 空间）
- 数组无序，且不能排序 → 哈希表

## 1.7 双指针思维的四步框架

无论哪种形态，解决双指针问题都遵循这四步：

```
1. 定义指针语义
   "left 指向什么？right 指向什么？"
   "slow 维护的不变量是什么？"

2. 确定移动条件
   "什么时候移动 left？什么时候移动 right？"
   "移动的方向是什么？步长是多少？"

3. 处理相遇/终止
   "两指针何时停止？终止条件是 left < right 还是 left <= right？"

4. 提取答案
   "答案是指针本身？还是窗口大小？还是某个累积值？"
```

这四步贯穿本教程所有章节，是建立解题直觉的核心框架。

## 1.8 本章小结

| 概念 | 含义 |
|------|------|
| 对撞指针 | 两端向中间逼近，适合有序数组 |
| 同向双指针 | 两指针同向，不同速度，适合子数组问题 |
| 快慢指针 | 步差思想，适合链表 |
| 单调性 | 双指针成立的数学前提 |
| O(n) 的原因 | 每个元素至多被访问常数次 |
