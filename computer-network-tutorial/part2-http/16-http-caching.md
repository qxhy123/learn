# 第16章：HTTP 缓存

## 本章目标

- 理解缓存为什么能减少延迟和带宽消耗
- 认识强缓存与协商缓存这两个核心思路
- 明白 `Cache-Control`、`ETag`、`Last-Modified`、`304` 的基本作用
- 能从浏览器角度理解一次缓存命中和一次重新验证

## 先讲结论

HTTP 缓存的核心目标很简单：

- 能不重新下载，就尽量不重新下载
- 即使需要确认，也尽量只确认“资源变没变”，而不是把整个内容再传一遍

因此浏览器缓存常见地分成两条思路：

- **强缓存**：本地副本还在有效期内，直接使用，不发请求或不取新内容
- **协商缓存**：浏览器去问服务器“资源有变化吗”，没变就返回 `304 Not Modified`

这套机制能显著提升页面加载速度，也能降低服务器和网络压力。

## 核心概念

### 1. 为什么缓存很重要

一个网页通常会加载很多静态资源：

- CSS
- JavaScript
- 图片
- 字体

如果用户每次刷新页面都把这些内容完整重新下载：

- 页面会更慢
- 流量浪费更多
- 服务器压力更大

而实际上，很多资源在一段时间内根本不会变，所以非常适合缓存。

### 2. 强缓存：先相信本地副本还能用

强缓存的思路是：

- 服务器提前告诉浏览器，这份资源在多久内可以直接使用
- 在有效期内，浏览器优先使用本地缓存副本

常见控制字段是：

- `Cache-Control`
- `Expires`

其中现代系统更常用 `Cache-Control`。

例如：

`Cache-Control: max-age=3600`

可以理解为：这份资源在 3600 秒内都可以直接复用。

### 3. 协商缓存：确认是否变化

如果强缓存过期了，或者策略要求重新确认，浏览器就会发起协商缓存。

这时浏览器不会直接盲目重下，而是带上资源标识去问服务器。

常见方式有：

- `ETag` / `If-None-Match`
- `Last-Modified` / `If-Modified-Since`

如果服务器判断资源没变，就返回：

- `304 Not Modified`

这样浏览器继续使用本地副本即可，不需要再传完整正文。

### 4. `ETag` 和 `Last-Modified` 各自在表达什么

可以先用最直观的方式理解：

- `Last-Modified`：这个资源上次修改时间是什么时候
- `ETag`：这个资源当前版本的一个标识

`ETag` 往往能更精细地表示“内容版本”，而 `Last-Modified` 更像是基于时间的粗粒度判断。

很多系统会同时使用二者，让缓存判断更稳妥。

## 过程拆解

### 场景 1：强缓存命中

第一次访问资源时，服务器返回：

```http
HTTP/1.1 200 OK
Cache-Control: max-age=600
Content-Type: text/css

body { color: #333; }
```

浏览器保存这份 CSS 后，在接下来 600 秒内再次访问同一资源时，通常会直接使用本地副本。

这时关键点是：

- 不必重新下载正文
- 页面加载会更快

### 场景 2：协商缓存命中

第一次请求时，服务器返回：

```http
HTTP/1.1 200 OK
ETag: "v2-abc123"
Content-Type: application/javascript

console.log("hello")
```

之后浏览器再次请求时，可以带上：

```http
GET /app.js HTTP/1.1
Host: example.com
If-None-Match: "v2-abc123"
```

如果服务器发现资源没变，就返回：

```http
HTTP/1.1 304 Not Modified
```

于是浏览器继续用本地缓存的 `app.js` 内容。

## 常见误区

### 误区 1：缓存就是浏览器偷偷存一下文件

不对。HTTP 缓存不是随便存，而是客户端和服务器共同约定的一套策略机制。

### 误区 2：返回 `304` 就说明没有缓存

不对。恰恰相反，`304` 说明浏览器本地已经有缓存副本，只是这次又向服务器确认了一下是否还能继续使用。

### 误区 3：缓存只和前端有关

不对。缓存策略通常需要服务端正确设置头部，也会影响 CDN、代理、中间层等多个环节。

## 小结

- HTTP 缓存的目标是减少重复传输和不必要的延迟
- 强缓存强调“本地副本在有效期内直接用”
- 协商缓存强调“向服务器确认资源是否变化”
- `Cache-Control`、`ETag`、`Last-Modified`、`304` 是理解缓存的核心关键词
