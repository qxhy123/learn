# 第12章：URL、URI、方法与状态码

## 本章目标

- 分清 URL 和 URI 的常见区别与联系
- 认识 HTTP 方法表达的操作语义
- 理解状态码为什么重要，以及常见分类怎么读
- 能读懂一个最基本的 HTTP 请求与响应

## 先讲结论

这一章的核心可以浓缩成四句话：

- **URI** 是更大的概念，用于标识资源
- **URL** 是 URI 的常见子集，强调资源在哪里、怎么访问
- **HTTP 方法** 表示客户端想对资源做什么
- **状态码** 表示服务器对这次请求处理得怎么样

学 HTTP，如果看不懂 URL、方法和状态码，就很难真正读懂浏览器和服务器之间在说什么。

## 核心概念

### 1. URI 与 URL

可以先用最实用的方式理解：

- **URI**：统一资源标识符，重点在“标识资源”
- **URL**：统一资源定位符，重点在“资源的位置和访问方式”

日常开发里，大家很多时候会把浏览器地址栏里的东西直接叫 URL，这样理解完全没问题。

例如：

`https://example.com:443/docs/index.html?lang=zh#intro`

这里通常可以拆成：

- 协议：`https`
- 主机：`example.com`
- 端口：`443`
- 路径：`/docs/index.html`
- 查询参数：`lang=zh`
- 片段：`#intro`

### 2. HTTP 方法

HTTP 方法描述客户端想干什么。常见方法包括：

- `GET`：获取资源
- `POST`：提交数据
- `PUT`：整体更新资源
- `PATCH`：部分更新资源
- `DELETE`：删除资源

方法本质上是在表达**请求语义**，而不只是一个字符串。

### 3. 状态码

状态码是服务器对请求处理结果的简洁表达。

最常见的分类方式：

- `1xx`：信息性状态
- `2xx`：成功
- `3xx`：重定向
- `4xx`：客户端错误
- `5xx`：服务器错误

几个高频状态码：

- `200 OK`：请求成功
- `201 Created`：资源创建成功
- `301 Moved Permanently`：永久重定向
- `302 Found`：临时重定向
- `400 Bad Request`：请求格式有问题
- `401 Unauthorized`：未认证
- `403 Forbidden`：已识别身份但无权限
- `404 Not Found`：资源不存在
- `500 Internal Server Error`：服务器内部错误
- `503 Service Unavailable`：服务暂时不可用

### 4. 为什么状态码不能只看数字名字

初学者容易死背数字，但真正重要的是语义。

例如：

- `401` 不等于“没权限”，它更强调“缺少有效身份认证”
- `403` 更接近“服务器理解你是谁，但不允许你做这件事”
- `404` 不一定说明服务器坏了，只是目标资源没找到

## 示例

### 示例 1：一个简单请求

```http
GET /articles/123?lang=zh HTTP/1.1
Host: example.com
Accept: application/json
```

可以这样读：

- 方法：`GET`，表示获取资源
- 路径：`/articles/123`
- 查询参数：`lang=zh`
- 协议版本：`HTTP/1.1`
- `Host` 表示目标主机
- `Accept` 表示希望收到 JSON 类型响应

### 示例 2：一个简单响应

```http
HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 27

{"id":123,"title":"网络基础"}
```

可以这样读：

- 状态码 `200` 表示请求成功
- `Content-Type` 表示响应体是 JSON
- `Content-Length` 表示响应体字节长度
- 空行之后是响应体内容

## 常见误区

### 误区 1：URL 和 URI 完全一样

严格来说不完全一样，但在很多工程语境里把日常地址称为 URL 是完全常见且可接受的。

### 误区 2：POST 就一定是“新增”，GET 就一定绝对安全

不应机械背诵。方法有推荐语义，但最终仍要结合服务器设计理解。尤其 GET 应尽量只读，但如果服务端设计不当，也可能产生副作用。

### 误区 3：4xx 和 5xx 都只是“请求失败”

不够精确。4xx 更多表示客户端请求本身有问题，5xx 更多表示服务器处理过程中出了问题。

## 小结

- URL/URI 帮助我们标识和定位资源
- HTTP 方法表达对资源的操作意图
- 状态码用简洁方式表达服务器处理结果
- 真正会读 HTTP，不是会背术语，而是能把一段请求和响应解释成人话
