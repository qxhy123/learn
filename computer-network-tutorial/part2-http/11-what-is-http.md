# 第11章：什么是 HTTP

## 本章目标

- 理解 HTTP 在整个网络协议栈中的位置
- 明白 HTTP 的核心是“规定客户端和服务器如何交换资源”
- 认识请求、响应、头部、消息体这些基础概念
- 建立对“无状态协议”的初步理解

## 先讲结论

HTTP 是 HyperText Transfer Protocol，即超文本传输协议。

它最核心的作用不是“负责上网”，而是：

- 规定客户端如何向服务器提出资源请求
- 规定服务器如何返回结果
- 规定双方交换消息时的大致格式和语义

也就是说，HTTP 关心的是“你想要什么资源、我返回什么结果”，而不是底层数据包如何跨网络抵达。

## 核心概念

### 1. HTTP 属于应用层协议

在分层模型里，HTTP 位于应用层。

它通常依赖下层协议完成传输，例如：

- HTTP/1.1 常见地跑在 TCP 之上
- HTTP/2 也通常跑在 TCP 之上
- HTTP/3 则基于 QUIC，而 QUIC 又建立在 UDP 之上

所以 HTTP 本身不负责 IP 路由、MAC 转发、丢包重传这些事，它只负责定义应用语义。

### 2. 请求与响应

HTTP 采用典型的请求-响应模式：

- 客户端发出请求（Request）
- 服务器返回响应（Response）

请求里通常包含：

- 方法
- 路径
- 请求头
- 可能的请求体

响应里通常包含：

- 状态码
- 响应头
- 响应体

### 3. 资源的思想

HTTP 的一个重要思想是“资源”。

一个 URL 往往对应服务器上的某个资源，例如：

- 一个 HTML 页面
- 一张图片
- 一个 JSON API 返回值
- 一个视频文件

客户端通过 HTTP 方法对资源进行获取、提交、更新或删除等操作。

### 4. HTTP 的无状态性

HTTP 常被称为无状态协议，意思是：

- 单独看一次请求时
- 服务器不一定天然记得你之前做过什么

这并不是说服务器完全不能记住用户，而是说：

- 协议本身不自动替你保存会话状态
- 如果需要登录态、购物车、会话等能力，需要借助 Cookie、Session、Token 等机制

## 过程拆解

### 浏览器请求网页时，HTTP 在做什么

假设你访问：

`http://example.com/index.html`

大致过程里，HTTP 负责的是：

1. 浏览器构造一条请求消息，例如“请把 `/index.html` 发给我”
2. 请求消息里会带上方法、路径、Host 等信息
3. 服务器收到后，根据这些信息找到对应资源
4. 服务器返回一个响应，告诉浏览器：
   - 请求成功还是失败
   - 返回的数据是什么类型
   - 内容本身是什么
5. 浏览器再根据响应内容决定如何渲染页面

所以，HTTP 更像是客户端与服务器之间的一套“交流规则”。

## 常见误区

### 误区 1：HTTP 就是传网页文本

不对。HTTP 最初和超文本密切相关，但今天它可以传输 HTML、JSON、图片、音视频、二进制文件等各种资源。

### 误区 2：HTTP 负责加密

不对。HTTP 本身不负责加密。加密通常由 HTTPS 中的 TLS 提供。

### 误区 3：HTTP 无状态就无法实现登录

不对。登录态完全可以实现，只是需要在 HTTP 之上配合 Cookie、Session、Token 等机制完成。

## 小结

- HTTP 是应用层协议，核心任务是定义客户端与服务器之间的请求-响应语义
- HTTP 关注资源、方法、状态码、消息格式，不负责底层路由与传输细节
- HTTP 的无状态性意味着状态管理要通过额外机制实现
- 理解 HTTP 的定位，后面学习 URL、方法、状态码和 HTTPS 会更自然
