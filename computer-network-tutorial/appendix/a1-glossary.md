# 附录 A1：术语表

## 使用说明

这一部分不是为了把术语变成新的背诵负担，而是为了帮助你在回看章节时，快速确认某个概念的大致意思。

如果某个术语你只能背出定义，却说不清它在流程中的位置，建议回到对应章节再看一遍。

## A

### ACK

Acknowledgment，确认应答。

在 TCP 中通常表示：

- 接收方告诉发送方，哪些数据已经成功收到
- 发送方据此决定是否继续发送、重传或滑动窗口

### ARP

Address Resolution Protocol，地址解析协议。

用于在局域网内根据 IP 地址找到对应的 MAC 地址，常见于主机要把数据发给同一网段主机或默认网关时。

## C

### CA

Certificate Authority，证书颁发机构。

在 HTTPS / TLS 体系中，CA 负责按照规则签发证书，把身份信息和公钥绑定起来，并通过签名提供可验证背书。

### Cache-Control

HTTP 响应头或请求头中的缓存控制字段。

用于表达资源是否可缓存、能缓存多久、是否必须重新验证等策略。

### ClientHello

TLS 握手中客户端发出的第一类关键消息。

常用于声明：

- 支持的协议版本
- 支持的密码参数
- 客户端随机数
- 某些扩展能力

### Cookie

浏览器保存并在后续请求中按规则携带的一小段状态数据。

常用于：

- 登录态标识
- 会话跟踪
- 个性化设置

## D

### DNS

Domain Name System，域名系统。

核心作用是把人类更容易记住的域名解析成机器通信所需的 IP 地址。

### DevTools

浏览器开发者工具。

在网络学习中，尤其常用的是 Network 面板，用于查看请求时序、状态码、缓存、Cookie、重定向等信息。

## E

### ETag

HTTP 中用于标识资源版本的一种响应头字段。

浏览器后续请求时可带上 `If-None-Match`，让服务器判断资源是否变化，从而支持协商缓存。

## F

### FIN

TCP 中用于表示“我这边数据发送完了，希望开始关闭连接”的控制标志。

它和 RST 不同，FIN 更偏向有序关闭，而不是异常中断。

## H

### Host

HTTP 请求头字段，用于指出请求目标主机名。

在 HTTP/1.1 中尤其关键，因为同一个服务器 IP 可能承载多个站点。

### HTTP

HyperText Transfer Protocol，超文本传输协议。

属于应用层协议，核心负责定义客户端与服务器如何交换请求和响应。

### HTTPS

可以简单理解为 HTTP 运行在 TLS 保护之上。

它不是新的业务语义协议，而是带有加密、完整性保护和身份认证能力的 HTTP 通信方式。

### HSTS

HTTP Strict Transport Security。

一种让浏览器记住“这个站点以后必须用 HTTPS 访问”的机制，用于降低从 HTTP 明文入口被降级或劫持的风险。

## I

### IP

Internet Protocol，网际协议。

负责主机到主机的寻址与跨网络转发，是网络层的核心协议之一。

### IP 地址

网络中用于标识主机或接口的地址。

浏览器访问网站时，真正用于底层网络寻址和转发的是 IP，而不是域名本身。

## M

### MAC 地址

链路层地址，常用于同一局域网或同一链路上的设备标识。

交换机转发以太网帧时主要看 MAC 地址，而跨网络路由则主要看 IP 地址。

### mTLS

Mutual TLS，双向 TLS。

不仅服务器向客户端出示证书，客户端也向服务器出示证书，常用于服务间通信或高安全要求场景。

## N

### NAT

Network Address Translation，网络地址转换。

常见于家庭路由器、企业网络和云环境，用于在不同地址空间之间转换 IP / 端口映射。

## O

### OSI

开放系统互连参考模型。

它更像一个教学和分析框架，用于帮助理解分层思想；真实互联网工程中更常直接使用 TCP/IP 协议族视角。

## P

### PATCH

HTTP 方法之一，常用于表示对资源进行部分更新。

### PFS

Perfect Forward Secrecy，前向保密性。

强调即使长期私钥未来泄露，也不应导致过去会话内容被统一回溯解密。

### PKI

Public Key Infrastructure，公钥基础设施。

围绕公钥信任建立的一整套体系，通常包括证书、CA、信任库、签发与吊销机制等。

### Port

端口。

用于在同一台主机上区分不同应用进程或网络服务，例如 HTTP 常见使用 80，HTTPS 常见使用 443。

## Q

### QUIC

一种建立在 UDP 之上的现代传输协议。

HTTP/3 运行在 QUIC 之上，目标之一是减少握手延迟并改进传输体验。

## R

### RST

TCP 中用于异常重置连接的控制标志。

如果看到连接被 RST，往往意味着对端或中间设备直接中止了连接，而不是正常结束。

### RTT

Round-Trip Time，往返时延。

表示一个请求从发送出去到收到对方响应所经历的大致往返时间，对握手速度和页面加载体验影响明显。

## S

### SAN

Subject Alternative Name，证书中的主题备用名称字段。

现代浏览器主要根据 SAN 中列出的域名来判断证书是否适用于当前访问的主机名。

### Session

常见语境下指服务端维护的会话状态。

浏览器通常借助 Cookie 携带会话标识，服务端再根据这个标识找到对应登录态或用户上下文。

### SNI

Server Name Indication。

TLS 握手中的一个扩展，用于让客户端在握手早期就告诉服务器自己想访问哪个主机名，以便服务器返回正确证书。

### Socket

套接字。

可以把它理解成应用程序与操作系统网络栈之间进行网络通信的一种编程接口抽象。

### Status Code

HTTP 状态码。

用于简洁表达服务器对一次请求的处理结果，例如 `200`、`404`、`500`。

### SYN

TCP 中用于发起连接建立的控制标志。

三次握手的第一步通常由客户端发送 SYN 开始。

## T

### TCP

Transmission Control Protocol，传输控制协议。

属于传输层，提供面向连接、尽量可靠、有序的字节流传输能力。

### TLS

Transport Layer Security，传输层安全协议。

HTTPS 的核心安全能力主要来自 TLS，它负责提供机密性、完整性和身份认证。

### Token

在 Web 语境中，通常指某种可用于证明身份或携带授权信息的令牌。

它常被放在 HTTP 头部或 Cookie 中传输，但不是 HTTP 协议自动自带的能力。

### TTL

Time To Live。

在不同上下文中含义略有不同。对 DNS 学习而言，通常表示解析记录的缓存生存时间；对 IP 而言，则表示包可继续转发的跳数限制。

## U

### UDP

User Datagram Protocol，用户数据报协议。

属于传输层，特点是无连接、开销较低、不保证可靠和有序，常用于 DNS、实时音视频、QUIC 等场景。

### URI

Uniform Resource Identifier，统一资源标识符。

更强调“标识资源”这一点，是比 URL 更大的概念。

### URL

Uniform Resource Locator，统一资源定位符。

日常最常见的就是浏览器地址栏里的地址，它强调资源的位置和访问方式。

## W

### Wireshark

常用的抓包分析工具。

适合用来观察 DNS、TCP、TLS、HTTP 等流量在包级别上的真实表现。

## 小结

- 术语表的价值在于帮助你快速回忆概念，不是替代章节理解
- 如果某个术语你只能背定义，却讲不清它在流程中的作用，建议回到对应章节重新串联
- 真正掌握术语的标志，不是会背英文全称，而是知道它在真实网络流程里做什么
