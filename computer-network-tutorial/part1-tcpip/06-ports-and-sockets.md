# 第6章：端口与 Socket

## 本章目标

- 理解端口为什么存在，以及它解决了什么问题
- 区分“主机地址”和“进程入口”的不同角色
- 认识 socket 作为网络编程抽象接口的含义
- 能解释“IP + 端口”为什么能定位到具体网络服务

## 先讲结论

IP 地址只能找到一台主机，但一台主机上通常运行着很多网络程序。

因此还需要端口来回答：

- 这份数据到了这台机器以后
- 应该交给哪个应用进程处理

而 socket 可以理解成操作系统提供给程序的“网络通信接口”或“通信端点抽象”。

## 核心概念

### 1. 端口是什么

端口是传输层使用的编号，用来标识主机上的不同通信入口。

比如：

- 同一台服务器上可以同时运行 Web 服务、数据库、SSH 服务
- 它们都共用这台机器的 IP
- 但通过不同端口区分不同服务

所以：

- **IP 定位主机**
- **端口定位主机上的具体服务或进程入口**

### 2. 常见端口号

有些端口是大家约定俗成的“知名端口”：

- HTTP：80
- HTTPS：443
- SSH：22
- DNS：53

这并不意味着服务只能跑在这些端口，而是这些端口是默认和常见选择。

### 3. Socket 是什么

从编程角度看，socket 是操作系统暴露出来的网络接口。

程序通常不是直接“操作 IP 包和网卡”，而是：

- 创建 socket
- 绑定地址和端口
- 连接对方或监听连接
- 通过 socket 读写数据

因此，socket 更接近“编程接口和通信端点”，不是单纯一个端口号。

### 4. 什么叫一个连接

在 TCP 中，常见用四元组区分一条连接：

- 源 IP
- 源端口
- 目标 IP
- 目标端口

这意味着：

- 一台服务器即使只监听一个端口，如 443
- 也能同时服务很多客户端连接
- 因为每个客户端的源 IP / 源端口组合通常不同

## 过程拆解

### 浏览器访问网站时，端口和 socket 怎么参与

假设浏览器访问：

`https://example.com`

大致过程：

1. 浏览器先得到目标服务器 IP
2. 浏览器准备连接目标端口 `443`
3. 操作系统为浏览器分配一个临时源端口，例如 `53124`
4. 浏览器通过客户端 socket 发起连接
5. 服务器在 `443` 端口上的监听 socket 接收到请求
6. 服务器为这条连接建立对应的已连接 socket，后续通过它与客户端收发数据

于是，一条连接可以表示为：

`客户端IP:53124 -> 服务器IP:443`

这就是为什么很多浏览器标签页可以同时访问同一网站，而服务器仍能区分它们。

## 常见误区

### 误区 1：端口就是进程本身

不对。端口是进程对外暴露的通信入口标识，不等于进程本身。

### 误区 2：一个端口只能服务一个客户端

不对。服务器监听一个端口后，可以同时处理来自很多客户端的连接。

### 误区 3：socket 就是 IP 加端口

不完全对。IP 和端口是 socket 标识的一部分，但 socket 更强调操作系统中的通信对象和编程接口。

## 小结

- IP 负责找到主机，端口负责把数据交给主机上的具体服务
- socket 是程序进行网络通信时使用的接口和端点抽象
- TCP 连接常用四元组唯一标识
- 学懂端口和 socket，后面理解 HTTP 服务、数据库连接、负载均衡会更容易
