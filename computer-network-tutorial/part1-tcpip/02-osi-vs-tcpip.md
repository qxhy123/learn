# 第2章：OSI 模型与 TCP/IP 模型

## 本章目标

- 了解 OSI 七层模型和 TCP/IP 模型的基本结构
- 明白两种模型分别属于“理论参考”和“工程实践”层面
- 能把常见协议放到合适的层中理解
- 理解教材为什么总讲 OSI，但现实世界主要运行 TCP/IP

## 先讲结论

学习网络时，经常会同时看到 OSI 模型和 TCP/IP 模型。最重要的结论是：

- **OSI 更像教学和分析用的参考模型**
- **TCP/IP 更像真实互联网长期采用的协议体系**
- 两者不是互相打架，而是**一个偏理论分层，一个偏工程落地**

所以，学 OSI 是为了建立清晰框架；学 TCP/IP 是为了理解现实网络如何真正工作。

## 核心概念

### 1. OSI 七层模型

OSI 通常分为七层：

1. 物理层
2. 数据链路层
3. 网络层
4. 传输层
5. 会话层
6. 表示层
7. 应用层

它的优点是划分细、概念清楚，适合教学时说明“不同问题由不同层处理”。

### 2. TCP/IP 模型

TCP/IP 常见教材版本分为四层，也有资料拆成五层：

- 应用层
- 传输层
- 网络层（或互联网层）
- 网络接口层（有时再拆成链路层、物理层）

真实互联网中常见协议基本都可以放进这套体系：

- 应用层：HTTP、DNS、SMTP
- 传输层：TCP、UDP
- 网络层：IP、ICMP
- 链路层：以太网、Wi‑Fi、ARP

### 3. 两者如何对应

可以粗略这样理解：

- OSI 的 **应用层 + 表示层 + 会话层**，大致对应 TCP/IP 的 **应用层**
- OSI 的 **传输层**，对应 TCP/IP 的 **传输层**
- OSI 的 **网络层**，对应 TCP/IP 的 **网络层**
- OSI 的 **数据链路层 + 物理层**，大致对应 TCP/IP 的 **网络接口层**

也就是说，TCP/IP 把 OSI 的上三层和下两层做了更工程化的合并。

### 4. 为什么现实世界更常讲 TCP/IP

因为互联网不是按 OSI 全套协议跑起来的，而是在 TCP/IP 协议族基础上发展壮大的。

例如当你访问网站时，真实用到的通常是：

- HTTP/HTTPS
- TCP 或 UDP/QUIC
- IP
- 以太网 / Wi‑Fi

这套组合显然更贴近 TCP/IP 模型，而不是严格照搬 OSI 七层。

## 过程拆解

### 以“浏览器访问网页”为例做层次映射

当浏览器访问 `https://example.com` 时：

- **应用层**：浏览器构造 HTTP 请求；DNS 先解析域名
- **传输层**：通常由 TCP 提供可靠传输
- **网络层**：IP 负责把分组从本机送到目标服务器
- **链路/网络接口层**：以太网或 Wi‑Fi 负责当前链路上的实际发送

如果用 OSI 的视角：

- HTTP 仍属于应用层范畴
- 数据编码格式可以看作表示层相关内容
- 会话管理可放入会话层视角理解
- TCP 对应传输层
- IP 对应网络层
- 以太网/Wi‑Fi 对应数据链路层和物理层

这说明 OSI 有助于“分析得更细”，TCP/IP 更适合“描述真实协议栈”。

## 常见误区

### 误区 1：OSI 已经过时，所以不用学

不对。OSI 虽然不是互联网实际运行时的完整协议体系，但它在教学、排障、架构分析里仍然非常有价值。

### 误区 2：TCP/IP 和 OSI 完全一一对应

不对。它们只能大致映射，不能机械地逐层硬对齐。

### 误区 3：TCP/IP 一定只有四层

教材不同，常见有四层和五层说法。关键不是死记层数，而是理解职责边界。

## 小结

- OSI 模型更偏参考框架，TCP/IP 模型更偏现实实现
- OSI 七层更细，TCP/IP 分层更工程化
- 学 OSI 有助于建立结构化认知，学 TCP/IP 有助于理解真实互联网协议
- 最有效的学习方式，是能把真实协议和分层职责对应起来
