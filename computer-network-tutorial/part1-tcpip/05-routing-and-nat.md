# 第5章：路由与 NAT

## 本章目标

- 理解“路由”解决的是跨网络转发问题
- 知道默认网关、下一跳、路由表的基本含义
- 理解 NAT 为什么会在家庭网络和云环境中大量出现
- 能从“内网访问外网”的角度解释 NAT 的工作过程

## 先讲结论

路由和 NAT 经常一起出现，但它们解决的问题并不一样：

- **路由**：决定数据包下一步该往哪里走
- **NAT**：在转发过程中修改地址或端口，让多个内网设备共享较少的公网地址

可以这样记：

- 路由回答“走哪条路”
- NAT 回答“用哪个对外身份出去”

## 核心概念

### 1. 什么是路由

路由就是根据目标 IP 地址，决定把包转发给哪个下一跳。

主机和路由器都会参考路由信息：

- 主机通常有本地直连路由和默认路由
- 路由器通常维护更复杂的路由表

当目标不在本地子网时，主机就会把包交给默认网关，由网关继续转发。

### 2. 路由表在干什么

路由表里通常包含：

- 目标网段
- 子网掩码或前缀长度
- 下一跳
- 出接口

设备拿到一个目标 IP 后，会从路由表里找最匹配的项，然后决定如何发送。常见原则是：**最长前缀匹配**。

### 3. 什么是 NAT

NAT 是 Network Address Translation，即网络地址转换。

最常见的是家庭路由器做的源地址转换：

- 内网设备使用私有 IP，例如 `192.168.x.x`
- 出公网时，路由器把源 IP 改成自己的公网 IP
- 如果很多内网设备同时上网，还会配合端口号区分不同连接

### 4. 为什么需要 NAT

NAT 常见的原因有两个：

1. **节省公网 IPv4 地址**
2. **让内网地址结构更灵活**

因此，家里很多设备虽然都能上网，但在公网看来，可能都像是同一个公网 IP 发出来的。

## 过程拆解

### 一个内网设备访问外部网站时会发生什么

假设：

- 你的电脑：`192.168.1.10:52341`
- 家用路由器公网 IP：`203.0.113.8`
- 目标网站：`93.184.216.34:80`

通信过程大致是：

1. 电脑构造一个发往 `93.184.216.34:80` 的 TCP 包
2. 电脑发现目标不在本地子网，于是把包交给默认网关
3. 家用路由器收到后，执行 NAT：
   - 源 IP 从 `192.168.1.10` 改成 `203.0.113.8`
   - 源端口可能也会改成一个新的公网侧端口
4. 路由器把转换关系记在 NAT 表里
5. 外部网站把响应发回 `203.0.113.8:某端口`
6. 路由器查 NAT 表，知道这其实对应内网电脑 `192.168.1.10:52341`
7. 路由器再把响应转回内网设备

这个过程说明：

- 路由决定“包应该先发给谁”
- NAT 决定“包出去时地址怎么改，回来时怎么还原”

## 常见误区

### 误区 1：有 NAT 就不需要路由

不对。NAT 不是路由的替代品。没有路由，包连下一跳都找不到；没有 NAT，只是很多内网私有地址无法直接在公网中使用。

### 误区 2：NAT 天然等于安全机制

不完全对。NAT 会让外部直接主动访问内网变难，但它本质上是地址转换技术，不等同于完整的防火墙和安全策略。

### 误区 3：所有网络都必须依赖 NAT

不对。在 IPv6 环境、某些企业网络或数据中心架构里，也可能不依赖传统 NAT。

## 小结

- 路由负责根据目标 IP 决定转发路径
- NAT 负责在转发过程中做地址或端口转换
- 主机访问外网时，通常先走默认网关，再由网关执行路由和可能的 NAT
- 理解路由与 NAT 的区别，是理解“为什么家里多台设备能共享上网”的关键
