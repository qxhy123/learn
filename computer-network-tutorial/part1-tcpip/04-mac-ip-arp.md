# 第4章：MAC、IP 与 ARP

## 本章目标

- 分清 MAC 地址、IP 地址、ARP 各自解决什么问题
- 理解“同一局域网通信”和“跨网络通信”的差异
- 知道主机为什么需要默认网关
- 能解释发送数据时为什么有时找目标主机 MAC，有时找网关 MAC

## 先讲结论

可以把这三个概念先粗略记成一句话：

- **IP 地址**：表示数据最终要去的主机
- **MAC 地址**：表示当前这一跳要发给哪块网卡
- **ARP**：在局域网里把 IP 地址解析成 MAC 地址

所以，IP 更像“最终目的地地址”，MAC 更像“当前投递这一站的收件地址”。而 ARP 就是帮你在本地网络里问一句：“这个 IP 对应的 MAC 是谁？”

## 核心概念

### 1. MAC 地址解决的是链路层交付

MAC 地址工作在链路层，主要用于同一局域网中的设备识别。

例如：

- 交换机看的是 MAC 地址
- 网卡接收帧时，也会看目标 MAC 是否是自己

MAC 地址通常只在**当前链路、当前广播域**内有意义，不能靠它跨整个互联网转发。

### 2. IP 地址解决的是网络层寻址

IP 地址工作在网络层，用于表示一台主机在网络中的位置。

路由器转发数据时，重点看的是：

- 目标 IP 是谁
- 下一跳该往哪走

因此，IP 是端到端通信的核心地址信息。

### 3. ARP 的作用

ARP 的全称是地址解析协议。它主要做一件事：

**已知某个局域网内的 IP 地址，询问对应的 MAC 地址是什么。**

典型流程是：

1. 主机先查本地 ARP 缓存
2. 如果没有，就发送 ARP 广播请求
3. 拥有该 IP 的设备回复自己的 MAC 地址
4. 主机把结果记入 ARP 缓存，后续一段时间直接使用

### 4. 为什么跨网通信时不是找“对方主机的 MAC”

这是初学者最容易混淆的地方。

如果目标主机和你**不在同一个子网**，你的主机并不会去找对方主机的 MAC，而是去找：

- **默认网关的 MAC 地址**

因为第一跳不是直接到远端主机，而是先交给路由器。路由器再继续往后转发。

## 示例

### 示例 1：同一局域网通信

假设：

- A：`192.168.1.10`
- B：`192.168.1.20`
- 两者都在同一个局域网

A 要给 B 发数据时：

1. A 发现目标 IP `192.168.1.20` 与自己在同一子网
2. A 查询 ARP 缓存，看是否知道 B 的 MAC
3. 如果不知道，就广播 ARP 请求
4. B 回复自己的 MAC
5. A 把数据帧直接发给 B 的 MAC

### 示例 2：跨网络通信

假设：

- A：`192.168.1.10`
- 网关：`192.168.1.1`
- 目标服务器：`8.8.8.8`

A 要访问 `8.8.8.8` 时：

1. A 发现目标 IP 不在本地子网
2. A 决定把数据先交给默认网关
3. A 通过 ARP 查询的是 **网关 `192.168.1.1` 的 MAC**
4. 以太网帧的目标 MAC 写成网关 MAC
5. 但 IP 头里的目标 IP 仍然是 `8.8.8.8`

这说明：

- **MAC 看当前一跳**
- **IP 看最终目的地**

## 常见误区

### 误区 1：知道目标 IP，就一定知道目标 MAC

不对。只有当目标在同一局域网，或者你要找的是下一跳网关时，ARP 才能帮你解析对应的 MAC。

### 误区 2：MAC 地址能跨互联网定位设备

不对。互联网上的跨网转发主要依赖 IP 和路由，不是依赖远端 MAC。

### 误区 3：ARP 是“互联网范围内”的地址解析

不对。ARP 主要发生在本地链路环境中，作用范围不是整个互联网。

## 小结

- MAC 负责当前链路上的交付，IP 负责端到端寻址
- ARP 的作用是在局域网内把 IP 解析成 MAC
- 同网段通信找目标主机 MAC，跨网通信通常找默认网关 MAC
- 学懂这一点，后面理解交换机、路由器、NAT 会顺很多
