# 第20章：流水线并行

> 张量并行是"横着切"每一层，流水线并行是"竖着切"模型的层。两者可以组合使用，覆盖更大规模的部署需求。

## 20.1 流水线并行原理

### 层间切分

```
完整模型 (32 层):
  Layer 0-31

PP=4 时:
  GPU 0: Layer  0- 7  (Stage 0)  + Embedding
  GPU 1: Layer  8-15  (Stage 1)
  GPU 2: Layer 16-23  (Stage 2)
  GPU 3: Layer 24-31  (Stage 3)  + LM Head
```

### 与张量并行的区别

| 维度 | 张量并行 (TP) | 流水线并行 (PP) |
|------|-------------|----------------|
| 切分方式 | 每层内部切分 | 层间切分 |
| 通信模式 | AllReduce（每层 2 次） | 点对点（stage 间传递激活） |
| 通信频率 | 非常频繁 | 较低（stage 间传一次） |
| 对带宽要求 | 高（需要 NVLink） | 低（PCIe 通常够用） |
| GPU 利用率 | 高（同时计算） | 有气泡（GPU 空闲等待） |

### 流水线气泡

```
非流水线执行 (串行):
  GPU 0: [Stage 0]→────────→
  GPU 1: ────────→[Stage 1]→────────→
  GPU 2: ─────────────────→[Stage 2]→────────→
  GPU 3: ──────────────────────────→[Stage 3]→
                                                  ↑ 3 个 GPU 大部分时间空闲

流水线执行 (micro-batch):
  GPU 0: [MB1][MB2][MB3][MB4]
  GPU 1:      [MB1][MB2][MB3][MB4]
  GPU 2:           [MB1][MB2][MB3][MB4]
  GPU 3:                [MB1][MB2][MB3][MB4]
         ↑        ↑                          ↑
      气泡(填充)  稳定状态              气泡(排空)
```

## 20.2 在 vLLM 中使用

### 基本配置

```python
from vllm import LLM

llm = LLM(
    model="meta-llama/Llama-3.1-70B-Instruct",
    pipeline_parallel_size=4,
)
```

### 命令行

```bash
vllm serve meta-llama/Llama-3.1-70B-Instruct \
    --pipeline-parallel-size 4
```

### TP + PP 组合

```bash
# 8 GPU: TP=4, PP=2
# 总并行度 = TP × PP = 8
vllm serve model \
    --tensor-parallel-size 4 \
    --pipeline-parallel-size 2
```

## 20.3 TP vs PP 选择策略

### 决策指南

```
单节点内 (NVLink 连接):
  优先使用 TP → 通信效率高

跨节点 (网络连接):
  优先使用 PP → 通信量小，对带宽要求低

混合场景 (多节点多卡):
  节点内 TP + 节点间 PP

示例: 2 节点 × 4 GPU/节点 = 8 GPU
  推荐: TP=4 (节点内), PP=2 (跨节点)
```

### 常见配置

| 模型大小 | GPU 数 | 推荐配置 |
|---------|--------|---------|
| 7-13B | 1 | TP=1 |
| 34-70B | 2-4 | TP=2-4 |
| 70B | 8 (单节点) | TP=8 或 TP=4×PP=2 |
| 70B | 8 (双节点) | TP=4×PP=2 |
| 405B | 16+ | TP=8×PP=2+ |

## 20.4 推理中的流水线并行

### 推理 vs 训练的区别

推理中流水线并行的气泡问题更严重：

```
训练: batch 大 → 可以用很多 micro-batch 填充流水线
推理: decode 时 batch 可能很小 → 气泡比例高

推理优化:
  1. 增大并发请求数（相当于增大 batch）
  2. 异步 pipeline scheduling
  3. 优先用 TP 减少对 PP 的依赖
```

## 本章小结

| 概念 | 要点 |
|------|------|
| 切分方式 | 按层切分到不同 GPU |
| 通信 | Stage 间点对点传递激活值 |
| 气泡 | GPU 等待前序 Stage 的空闲时间 |
| 适用场景 | 跨节点部署，带宽有限 |
| 与 TP 组合 | 节点内 TP + 节点间 PP |
