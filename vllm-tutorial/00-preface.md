# 前言：如何使用本教程

## 教程设计理念

高性能 LLM 推理不是"把模型丢到 GPU 上、调一个 API"就结束的事。真正掌握 vLLM 和 LLM 推理系统，需要同时建立三层理解：

1. **推理直觉**：为什么自回归生成天然慢？KV Cache 如何改变了计算模式？内存瓶颈到底在哪？
2. **系统直觉**：PagedAttention 解决了什么问题？连续批处理为什么能提升吞吐？调度器如何在延迟和吞吐之间权衡？
3. **工程直觉**：什么时候该量化？什么时候该上多 GPU？什么时候该先 benchmark 再决策？

本教程围绕这三层能力展开，遵循以下设计原则：

1. **从零开始，但不绕远路**：默认你没有 vLLM 经验，但也不会把教程变成 Transformer 入门课
2. **先理解设计动机，再学使用方法**：每个特性先讲"为什么需要"，再讲"怎么用"
3. **实验驱动**：重要概念尽量通过可运行的 Python 代码和 benchmark 说明
4. **贴近生产**：不仅讲离线推理，还讲在线服务、监控、调优、分布式和 K8s 部署
5. **中文优先**：尽量把术语说清，把直觉讲透，把坑点点明

## 为什么这套教程值得从头读完？

很多人使用 vLLM 的方式是：

- 看官方文档，copy 一段启动命令
- `vllm serve model_name` 跑起来
- 用 OpenAI SDK 调通
- 然后就上线了

结果通常是：

- 服务能跑，但不知道为什么突然变慢
- GPU 利用率看起来不高，但不知道瓶颈在哪
- 听说过 PagedAttention、连续批处理、投机解码，但不知道它们各自解决什么问题
- 量化后精度下降了，不知道该选 GPTQ 还是 AWQ 还是 FP8
- 多 GPU 部署时不知道该用张量并行还是流水线并行
- 生产环境出了 OOM，不知道如何调整内存参数

本教程的目标不是让你"会启动 vLLM"，而是让你在遇到 LLM 推理问题时，能系统地回答这些问题：

1. 当前瓶颈是 prefill 计算、KV Cache 内存、还是调度延迟？
2. 这个模型应该用什么量化方案？精度-速度-显存的权衡点在哪？
3. 这个场景适合投机解码还是前缀缓存？
4. 多 GPU 时应该怎么切分模型？通信开销是否可接受？
5. 监控指标中哪些值异常说明系统在退化？
6. 如何系统化地进行性能调优，而不是到处改参数碰运气？

## 章节结构

每章采用相对统一的结构，便于系统学习：

### 1. 学习目标
每章开头列出 5 个学习目标，帮助你在开始前明确重点。

### 2. 正文内容
正文一般包含 4-6 个部分，通常覆盖：
- 问题与动机
- 原理与设计
- 代码示例与 API 说明
- 常见误区与注意事项
- 工程实践建议

### 3. 本章小结
以表格或要点列表回顾本章的核心概念与关键结论。

### 4. 动手实验
每章都会设计至少一个实验，重点不是"跑出一个数字"，而是让你观察：
- 修改参数后行为有什么变化
- 不同配置之间的性能差异有多大
- 理论预期与实际结果是否一致
- 是否需要 benchmark 或 profiler 才能看清问题

### 5. 练习题
每章练习题分为三类：
- **基础题**：验证概念是否真正理解
- **实践题**：要求运行代码、修改配置或分析输出
- **思考题**：要求你解释设计决策、分析权衡或预测行为

### 6. 答案提示
完整答案汇总在附录中，鼓励你先独立思考，再查看要点。

## 代码约定

本教程示例主要使用 Python，并遵循以下约定：

```python
# 完整可运行的示例会标注"完整示例"
# --- 完整示例：离线推理 ---
from vllm import LLM, SamplingParams

llm = LLM(model="meta-llama/Llama-3.1-8B-Instruct")
params = SamplingParams(temperature=0.7, max_tokens=256)
outputs = llm.generate(["Hello, world!"], params)
print(outputs[0].outputs[0].text)
```

## 术语约定

本教程对关键术语采用以下处理方式：

| 英文术语 | 中文翻译 | 说明 |
|----------|----------|------|
| PagedAttention | PagedAttention（不翻译） | vLLM 核心技术，保留原名 |
| KV Cache | KV Cache（不翻译） | 业界通用术语 |
| Continuous Batching | 连续批处理 | 与"静态批处理"对照使用 |
| Prefill | 预填充 | 推理的第一阶段 |
| Decode | 解码 | 推理的第二阶段（逐 token 生成） |
| Speculative Decoding | 投机解码 | 也称"推测解码" |
| Prefix Caching | 前缀缓存 | 复用共享前缀的 KV Cache |
| Tensor Parallelism | 张量并行 | 跨 GPU 切分张量 |
| Pipeline Parallelism | 流水线并行 | 跨 GPU 切分层 |
| TTFT | 首 token 延迟 | Time To First Token |
| TPOT | token 间延迟 | Time Per Output Token |
| Throughput | 吞吐 | 单位时间处理的 token 数或请求数 |

## 关于版本

本教程基于 vLLM **v0.6.x - v0.8.x** 编写。vLLM 发展迅速，API 和特性可能随版本更新而变化。教程中会标注需要特定版本的功能，建议读者同时参考 [vLLM 官方文档](https://docs.vllm.ai/) 获取最新信息。
